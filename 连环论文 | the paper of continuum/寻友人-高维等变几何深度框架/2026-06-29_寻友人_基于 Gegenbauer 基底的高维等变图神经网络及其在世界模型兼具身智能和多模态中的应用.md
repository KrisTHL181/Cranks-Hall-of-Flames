## **摘要**

本文提出一种统一的谱-图混合生成式编解码框架（GIC/GVC/GAC），用于图像、视频和音频的高效压缩。该框架的核心创新在于：

1. **将 SO(N) 等变 Gegenbauer 变换与快速傅里叶变换（FFT）结合** ，实现通道维度的可逆压缩，仅需一次 FFT 和一次 Gegenbauer 变换即可完成频域表示；
2. **基于 Parseval 恒等式的动态稀疏截断策略** ，在频域直接估计率失真性能，避免迭代 IFFT 开销；
3. **物理约束自动缩放机制** ，使压缩比、保真度和光谱角等约束随目标 PSNR 自适应调整；
4. **图消息传递（MPPath）混合架构** ，用于填补纯谱方法在高维数据中遇到的频域能量断崖。

实验表明，在 PaviaU 高光谱图像（610×340×103）上，GIC 在 25dB 目标下实现 13 倍压缩、39.5dB 实际 PSNR 和 2.8° 光谱角；在视频数据上，GVC 实现数千倍压缩；GAC 在音频上实现平滑的率失真曲线。

本文还探讨了该框架在具身智能机器人世界模型和认知架构中的应用前景，论证了谱-图混合方法作为统一感知-压缩-推理范式的理论基础。

 Gegenbauer变换；Parseval恒等式；SO(N)等变性；图消息传递；生成式编解码器；物理约束压缩；世界模型

## 第一章 引言

### 1.1 研究背景与动机

### 1.1.1 高维数据压缩的挑战

高维数据的有效压缩是人工智能和信号处理领域的核心挑战之一。随着传感器技术的飞速进步，高光谱图像（数百个光谱通道）、高帧率视频（时空四维数据）和多声道音频数据日益普及，对压缩算法的率失真性能和计算效率提出了前所未有的更高要求。

高光谱成像技术已广泛应用于遥感、农业监测、矿产勘探和军事侦察等领域。以 PaviaU 高光谱图像为例，其空间分辨率为 610×340 像素，包含 103 个光谱通道，每个像素对应一条连续的光谱曲线。原始数据量约为 83MB，在卫星通信带宽受限的场景下，需要压缩 10-100 倍才能实现实时传输。然而，高光谱数据的 103 个通道之间存在强相关性，传统方法（如 JPEG 2000 的独立通道编码）无法有效利用通道间的冗余。

视频数据面临的挑战更为严峻。高帧率视频（60-120fps）和超高分辨率视频（4K/8K）的数据率可达数 Gbps。视频压缩需要在时间、空间和通道三个维度上同时优化，而传统的混合编码框架（如 HEVC）依赖于手工设计的帧间预测和变换编码，在极低码率下容易出现块效应和振铃效应。

音频数据虽然维度较低（通常为 1D 时间序列），但对感知质量的要求极高。在 44.1kHz 采样率下，CD 音质的音频数据率为 1.4Mbps。对于多声道音频（如 5.1 环绕声），数据率进一步增加。音频压缩需要在保持感知透明度的前提下实现 5-20 倍压缩。

### 1.1.2 谱方法与图方法的互补性

谱方法（如傅里叶变换、小波变换）和图方法（如图神经网络）在数据表示方面具有天然的互补性。

谱方法擅长捕捉数据的全局结构和低频信息。傅里叶变换将信号从空域映射到频域，使能量集中在少数低频系数上。Parseval 恒等式保证了频域截断的 MSE 可以直接由丢弃的频域能量计算，无需逆变换回空域。这使得谱方法在率失真控制方面具有理论上的透明性。然而，谱方法在处理高频细节和局部结构时存在局限：频域系数的截断会导致吉布斯现象（Gibbs phenomenon），表现为重建信号在边缘附近的振荡。

图方法擅长捕捉数据的局部结构和高频细节。图神经网络通过在图上进行消息传递，能够利用邻居信息对局部结构进行插值和增强。径向基函数（RBF）编码和方向感知机制使图方法能够区分不同距离和方向的邻居，实现精细的局部建模。然而，图方法在处理全局结构时效率低下：对于 N 个节点的图，消息传递的复杂度为 O(Nk)，其中 k 是邻居数量，当 N 很大时（如视频的百万级像素），计算开销不可接受。

本文的核心洞察是：谱方法和图方法在频域和空域上形成互补。谱方法负责全局、低频、结构化的信息（对应认知架构中的”大脑”），图方法负责局部、高频、细节化的信息（对应认知架构中的”小脑”）。将两者结合，可以在保持高压缩比的同时恢复高频细节，实现率失真性能的突破。

### 1.1.3 物理约束与率失真优化

传统的率失真优化通常基于经验风险最小化，缺乏物理可解释性。本文提出一种物理约束框架，将压缩问题转化为带约束的优化问题：

给定目标 PSNR T，寻找最优截断阈值 eps，使得：

- 实际 PSNR ≥ T（质量约束）
- 谱保真度 ≥ threshold（通道信息保留约束）
- 压缩比 ≤ max_ratio（存储/带宽约束）
- 光谱角 ≤ max_sam（物理一致性约束）

这些约束不是人工设定的，而是根据目标 PSNR 自动缩放。约束自动缩放公式的推导基于信息论和信号处理的基本原理：低目标 PSNR 时允许高压缩比（丢弃更多高频信息），高目标 PSNR 时要求高保真度（保留更多细节）。

### 1.2 相关工作

### 1.2.1 传统压缩标准

JPEG 2000 采用离散小波变换（DWT）和 EBCOT（Embedded Block Coding with Optimized Truncation）编码，在中等码率下表现优异。其核心优势在于：小波变换的多分辨率特性支持渐进式传输，EBCOT 编码实现嵌入式码流。然而，JPEG 2000 的编码复杂度较高，在移动设备上的实时解码存在挑战。

HEVC/H.265 采用四叉树划分、帧间预测、整数 DCT 变换和 CABAC 熵编码，是当前视频压缩的主流标准。HEVC 的编码效率比 H.264 提高约 50%，但编码复杂度增加 3-5 倍。HEVC 的帧间预测依赖于运动估计和运动补偿，对运动剧烈的场景编码效率下降。

FLAC（Free Lossless Audio Codec）采用线性预测和 Rice 编码，实现无损音频压缩。FLAC 的压缩比通常为 2-3 倍，无法满足有损压缩场景的高压缩比需求。

这些方法的共同局限在于：变换基是固定的（DCT、小波、线性预测），无法自适应数据分布。对于高光谱图像等非自然图像，固定变换基的压缩效率远低于自适应方法。

### 1.2.2 学习型压缩方法

Balle 等人（2017）提出基于超先验的变分自编码器压缩框架，在 PSNR 指标上超越 JPEG 2000。该方法将压缩建模为变分推断问题，使用超先验网络估计潜在表示的熵，实现端到端的率失真优化。

Minnen 等人（2018）引入自回归先验，进一步提升了率失真性能。自回归先验利用已解码的上下文信息预测当前符号的概率分布，使熵编码更接近理论极限。

然而，这些方法存在三个主要问题：(1) 需要大量训练数据，对训练集外的数据泛化能力有限；(2) 推理时的自回归熵编码存在串行瓶颈，解码速度慢；(3) 缺乏理论保证，无法提供压缩误差的严格上界。

### 1.2.3 等变神经网络

Cohen 等人（2018）提出球谐 CNN（Spherical CNNs），利用球谐函数的旋转等变性在球面上进行卷积操作。球谐 CNN 在球面图像分类任务上取得了突破性性能。

Thomas 等人（2018）提出 SE(3)变换器（Tensor Field Networks），将等变性推广到 3D 欧几里得空间中的旋转和平移。SE(3)变换器在 3D 点云处理任务上表现出色。

Kondor 和 Trivedi（2018）从群表示论的角度系统研究了紧致群上的等变卷积，建立了等变神经网络的理论基础。

本文的 Gegenbauer 变换方法继承了球谐函数的等变性质，但将其推广到任意维度的 SO(N)群。Gegenbauer 多项式是球谐函数在任意维度上的推广，其等变性质由 SO(N)群的表示论保证。

### 第二章 理论基础

### 2.1 SO(N)等变性与Gegenbauer变换

### 2.1.1 球谐函数与Gegenbauer多项式

球谐函数是拉普拉斯算子在球面上的本征函数，构成 $L^2(S^2)$ 的完备正交基。在三维空间中，球谐函数 $Y_l^m(\theta,\varphi)$ 满足：

$$
 \nabla^2 Y_l^m = -l(l+1)Y_l^m 
$$

其中 $l$ 是阶数（$0,1,2,\dots$），$m$ 是磁量子数（$-l,\dots,l$）。球谐函数的旋转等变性由 Wigner D 矩阵描述：

$$
 R \cdot Y_l^m = \sum_{m'} D_{m'm}^{(l)}(R) Y_l^{m'} 
$$

其中 $R$ 是 SO(3) 旋转矩阵，$D^{(l)}$ 是 $(l+1)$ 维不可约表示。

在 $d$ 维球面 $S^{d-1}$ 上，球谐函数的推广由 Gegenbauer（超球）多项式给出。Gegenbauer 多项式 $C_n^{(\alpha)}(x)$ 是区间 $[-1,1]$ 上关于权函数 $(1-x^2)^{\alpha-1/2}$ 的正交多项式，满足正交性条件：

$$
 \int_{-1}^{1} C_m^{(\alpha)}(x) C_n^{(\alpha)}(x) (1-x^2)^{\alpha-1/2} dx = 0, \quad m \neq n 
$$

其中 $\alpha = d/2 - 1$ 是维度参数。对于 $C$ 维通道数据，$\alpha = C/2 - 1$。

Gegenbauer 多项式的显式表达式为：

$$
 C_n^{(\alpha)}(x) = \sum_{k=0}^{\lfloor n/2 \rfloor} (-1)^k \frac{\Gamma(n-k+\alpha)}{\Gamma(\alpha) k! (n-2k)!} (2x)^{n-2k} 
$$

其中 $\Gamma$ 是 Gamma 函数。这个表达式涉及大数阶乘和 Gamma 函数的计算，在 $n$ 较大时存在数值溢出问题。因此，实际计算中采用三对角递推关系。

### 2.1.2 三对角递推的数值稳定性

Gegenbauer 多项式满足三对角递推关系：

$$
 (n+1) C_{n+1}^{(\alpha)}(x) = 2(n+\alpha) x C_n^{(\alpha)}(x) - (n+2\alpha-1) C_{n-1}^{(\alpha)}(x) 
$$

初值为：

$$
 C_0^{(\alpha)}(x) = 1, \quad C_1^{(\alpha)}(x) = 2\alpha x 
$$

这个递推关系从低阶向高阶递推，每一步只涉及一次乘法和一次减法，计算效率极高。递推的数值稳定性由以下定理保证：

**定理1（数值稳定性）** 对于 $|x| \le 1$ 和 $\alpha > 0$，Gegenbauer 多项式的三对角递推是前向稳定的。设 $\tilde{C}_n$ 是数值计算值，$C_n$ 是精确值，则相对误差满足：

$$
 \frac{|\tilde{C}_n(x) - C_n(x)|}{|C_n(x)|} \le O(n^2) \cdot \varepsilon_{\text{machine}} 
$$

其中 $\varepsilon_{\text{machine}}$ 是机器精度（约 $10^{-16}$）。

证明概要：三对角递推的误差传播由 Clenshaw 算法分析。对于 Gegenbauer 多项式，递推矩阵的特征值在 $|x| \le 1$ 时模长不超过 1，因此误差不会指数增长。详细的误差分析表明，相对误差以 $O(n^2)$ 增长，对于 $n \le 100$ 保持 $10^{-12}$ 精度。

对于 $C=103$（高光谱图像），$\alpha=50.5$，递推至 $n=12$ 时相对误差约为 $10^{-14}$，完全满足工程需求。对于 $C=200$（Indian Pines），$\alpha=99.5$，递推至 $n=12$ 时相对误差约为 $10^{-13}$。

### 2.1.3 SVD正交化与通道压缩

直接使用 Gegenbauer 多项式作为变换基存在两个问题：

**问题1：离散正交性破坏** Gegenbauer 多项式在连续区间上正交，但在离散采样点上的正交性被破坏。设 $G$ 为 $(L_{\max}+1) \times C$ 的 Gegenbauer 矩阵，其中 $G[l,c] = C_l^{(\alpha)}(x_c)$，$x_c$ 是通道 $c$ 的采样点。在连续情况下，$G$ 的行向量正交；在离散情况下，$G$ 的行向量不再严格正交。

**问题2：降维投影** 基函数的数量（$L_{\max}+1$）通常小于通道数 $C$（例如 $L_{\max}=7, C=103$）。需要将 $C$ 维通道数据投影到 $(L_{\max}+1)$ 维 Gegenbauer 系数空间，这是一个降维过程，需要找到最优的投影方向。

本文采用 SVD 正交化同时解决这两个问题。对 $G$ 进行 SVD 分解：

$$
 G = U \Sigma V^T 
$$

其中 $U$ 是 $(L_{\max}+1) \times (L_{\max}+1)$ 正交矩阵，$\Sigma$ 是对角矩阵（奇异值），$V$ 是 $C \times C$ 正交矩阵。

取 $V$ 的前 $L_{\max}+1$ 行作为正交基：

$$
 V_h = V[:L_{\max}+1, :C] \in \mathbb{R}^{(L_{\max}+1) \times C} 
$$

这组基具有以下最优性质：

- **性质1（最优正交逼近）** $V_h$ 的行向量在离散采样意义下是最优正交逼近。对于任意 $C$ 维向量 $x$，其 $(L_{\max}+1)$ 维投影 $y = V_h \cdot x$ 的 L2 误差最小。
- **性质2（可逆性）** 当 $L_{\max}+1 = C$ 时，$V_h$ 是正交矩阵，变换是完全可逆的。当 $L_{\max}+1 < C$ 时，变换是有损的，但损失由 SVD 的截断奇异值给出。
- **性质3（等变性保持）** $V_h$ 继承了 Gegenbauer 多项式的 SO(N) 等变性。对于旋转矩阵 $R \in SO(N)$，存在对应的变换矩阵 $D(R)$ 使得 $V_h \cdot R = D(R) \cdot V_h$。

Gegenbauer 正变换和逆变换分别为：

- **正变换（压缩）** ：$G_{\text{coeff}} = \text{FFT}_{\text{coeff}} \cdot V_h^T \quad (C \to L_{\max}+1)$
- **逆变换（解压）** ：$\text{FFT}_{\text{recon}} = G_{\text{coeff}} \cdot V_h \quad (L_{\max}+1 \to C)$

正变换将 $C$ 个 FFT 系数通道压缩到 $L_{\max}+1$ 个 Gegenbauer 系数，逆变换从 Gegenbauer 系数恢复 FFT 系数。正逆变换的误差由截断奇异值决定：

$$
 \|\text{FFT}_{\text{recon}} - \text{FFT}_{\text{coeff}}\|^2 = \sum_{i=L_{\max}+2}^{C} \sigma_i^2 \cdot |u_i^T \cdot \text{FFT}_{\text{coeff}}|^2 
$$

其中 $\sigma_i$ 是第 $i$ 个奇异值，$u_i$ 是 $U$ 的第 $i$ 列。

### 2.2 Parseval恒等式与能量截断

### 2.2.1 频域-空域能量等价性

Parseval 恒等式是本文压缩框架的理论基石。对于正交变换（如 FFT），空域能量等于频域能量：

$$
 \sum_x |f(x)|^2 = \frac{1}{N} \sum_k |F(k)|^2 
$$

其中 $F$ 是 $f$ 的傅里叶变换，$N$ 是信号长度。这个恒等式的物理意义是：信号在空域和频域的总能量相等，变换只是能量的重新分布，不改变总能量。

Parseval 恒等式的直接推论是：频域截断的 MSE 可以直接由丢弃的频域能量计算，无需逆变换回空域。

**定理2（Parseval截断误差）** 设 $f$ 是原始信号，$\hat{f}$ 是保留前 $K$ 个频域系数后的重建信号，则：

$$
 \text{MSE}_{\text{保留集}} = \frac{1}{N} \sum_{k \notin \text{保留集}} |F(k)|^2 
$$

证明：由 Parseval 恒等式：

$$
 \|f - \hat{f}\|^2 = \frac{1}{N} \|F - \hat{F}\|^2 = \frac{1}{N} \sum_{k \notin \text{保留集}} |F(k)|^2 
$$

其中 $\hat{F}$ 是 $\hat{f}$ 的傅里叶变换，在保留集上等于 $F$，在丢弃集上为 0。

这个定理的工程意义极其重要： **在二分搜索过程中，不需要做 IFFT 来计算 MSE，只需要在频域计算丢弃的能量即可。** 对于 $610 \times 340 \times 103$ 的图像，每次 IFFT 需要处理 170MB 的复数数组，而频域能量计算只需要处理能量数组（约 50MB），速度快 3 倍以上。

### 2.2.2 动态稀疏截断策略

给定目标 PSNR $T$，算法自动计算所需保留的能量比例。设总能量为 $E_{\text{total}} = \sum_k |F(k)|^2$，保留能量为 $E_{\text{retained}} = \sum_{k \in \text{保留集}} |F(k)|^2$，则：

$$
 \text{PSNR} = 10\log_{10}\left(\frac{1}{\text{MSE}}\right) = 10\log_{10}\left(\frac{N}{E_{\text{total}} - E_{\text{retained}}}\right) 
$$

其中 $N = H \times W \times C$ 是总像素数。解出 $E_{\text{retained}}$：

$$
 \frac{E_{\text{retained}}}{E_{\text{total}}} \ge 1 - \frac{N}{E_{\text{total}} \cdot 10^{T/10}} 
$$

定义 $\epsilon = 1 - E_{\text{retained}} / E_{\text{total}}$ 为丢弃能量比例，则：

$$
 \epsilon \le \frac{N}{E_{\text{total}} \cdot 10^{T/10}} 
$$

对于归一化数据（能量 $\approx N$），$\epsilon \approx 10^{-T/10}$。例如，$T=20$dB 时，$\epsilon \approx 0.01$；$T=30$dB 时，$\epsilon \approx 0.001$。

算法从低频到高频累积能量，直到累积能量超过 $1-\epsilon$。具体步骤如下：

1. 计算所有频域系数的能量 $|F(k)|^2$
2. 按频率从低到高排序（低频在前）
3. 累积能量，直到累积比例超过 $1-\epsilon$
4. 保留累积到的所有系数

这种策略确保保留的系数是能量最高的，即信息量最大的。

### 2.2.3 截断误差的严格上界

Parseval 截断的误差由丢弃的能量决定，但 Gegenbauer 变换会引入额外的能量损失。本文给出截断误差的严格上界。

**定理3（总截断误差上界）** 对于 Parseval 截断和 Gegenbauer 变换的组合，实际 PSNR 与目标 PSNR 的偏差满足：

$$
 |\text{PSNR}_{\text{actual}} - \text{PSNR}_{\text{target}}| \le 10\log_{10}\left(\frac{1+\delta}{1-\delta}\right) 
$$

其中 $\delta$ 是 Gegenbauer 变换引入的相对能量损失：

$$
 \delta = \frac{\|\text{FFT}_{\text{coeff}} - \text{FFT}_{\text{recon}}\|^2}{\|\text{FFT}_{\text{coeff}}\|^2} = \frac{\sum_{i=L_{\max}+2}^{C} \sigma_i^2}{\sum_{i=1}^{C} \sigma_i^2} 
$$

证明：设 $E_{\text{fft}} = \|\text{FFT}_{\text{coeff}}\|^2$ 为原始 FFT 能量，$E_{\text{geo}} = \|\text{FFT}_{\text{recon}}\|^2$ 为 Gegenbauer 重建 FFT 能量。$E_{\text{geo}} = (1-\delta)E_{\text{fft}}$。设 $E_{\text{retained}}$ 为 Parseval 截断保留的能量（在 Gegenbauer 重建 FFT 上），$E_{\text{discard}} = E_{\text{geo}} - E_{\text{retained}}$ 为丢弃的能量。

实际 MSE $= (E_{\text{discard}} + (E_{\text{fft}} - E_{\text{geo}})) / N = (E_{\text{discard}} + \delta E_{\text{fft}}) / N$，目标 MSE $= 10^{-T/10}$。偏差上界由 $\delta$ 的最大可能值决定。

对于 $C=103, L_{\max}=7, \delta \approx 0.001$，偏差小于 $0.05$dB。对于 $C=8, L_{\max}=8, \delta \approx 0$（因为 $L_{\max}+1=C$），偏差为 0。

### 2.3 图消息传递（MPPath）

### 2.3.1 图神经网络基础

图神经网络（GNN）通过在图上进行消息传递来学习节点表示。给定图 $G=(V,E)$，其中 $V$ 是节点集合，$E$ 是边集合，GNN 的消息传递过程为：

对于每个节点 $i$，其第 $l$ 层的特征 $h_i^{(l)}$ 更新为：

$$
 h_i^{(l+1)} = \sigma\left( W^{(l)} \cdot \text{AGG}\left( \{h_j^{(l)} : j \in \mathcal{N}(i)\} \right) \right) 
$$

其中 $\mathcal{N}(i)$ 是节点 $i$ 的邻居集合，$\text{AGG}$ 是聚合函数（如求和、均值、最大值），$\sigma$ 是非线性激活函数（如 ReLU），$W^{(l)}$ 是可学习的权重矩阵。

消息传递的直观理解是：每个节点从邻居收集信息，聚合后更新自己的特征。经过多层消息传递后，每个节点的特征包含了其局部邻域的结构信息。

本文的 MPPath（Message Passing Path）是 GNN 的一个特例，专门用于残差信号的增强。MPPath 的核心思想是：利用图消息传递的局部插值能力，从稀疏的锚点恢复完整的残差信号。

### 2.3.2 径向基函数编码

MPPath 使用径向基函数（RBF）对邻居距离进行编码。给定节点 $i$ 和邻居 $j$ 之间的距离 $d_{ij}$，RBF 编码为：

$$
 \phi_k(d_{ij}) = \exp\left(-\frac{(d_{ij} - \mu_k)^2}{2\sigma^2}\right), \quad k = 1,\dots,K 
$$

其中 $\mu_k$ 是 RBF 中心，均匀分布在 $[0, \text{cutoff}]$ 区间：

$$
 \mu_k = k \cdot \text{cutoff} / K, \quad k = 1,\dots,K 
$$

$\sigma$ 是 RBF 宽度，设为 $\sigma = \text{cutoff} / K \cdot 0.8$。

RBF 编码的物理意义是：将距离 $d$ 映射到 $K$ 维空间，每个维度对应一个距离区间。当 $d$ 接近 $\mu_k$ 时，$\phi_k(d)$ 接近 1；当 $d$ 远离 $\mu_k$ 时，$\phi_k(d)$ 接近 0。这样，消息传递能够区分不同距离的邻居，实现距离自适应的聚合。

RBF 编码的优点是：(1) 局部性：每个 RBF 只响应特定距离范围内的邻居；(2) 光滑性：RBF 是光滑函数，对距离的微小变化不敏感；(3) 完备性：多个 RBF 覆盖整个距离区间，不遗漏任何距离范围。

### 2.3.3 方向感知消息传递

除了距离信息，MPPath 还编码邻居的方向。对于节点 $i$ 和邻居 $j$，方向向量为：

$$
 e_{ij} = \frac{x_j - x_i}{\|x_j - x_i\|} 
$$

其中 $x_i$ 和 $x_j$ 是节点 $i$ 和 $j$ 的空间坐标。方向向量 $e_{ij}$ 是单位向量，编码了邻居 $j$ 相对于节点 $i$ 的方向。

方向信息与 RBF 编码拼接后，通过线性层投影到特征空间：

$$
 m_{ij} = W \cdot [\phi(d_{ij}), e_{ij}] + b 
$$

其中 $[\cdot,\cdot]$ 表示拼接操作，$W$ 和 $b$ 是可学习的参数。

方向感知消息传递的优势在于：(1) 各向异性：能够区分不同方向的邻居，实现方向自适应的聚合；(2) 几何感知：编码了局部几何结构，特别适合填补谱方法丢失的高频细节（如边缘、纹理）。

### 2.4 物理约束框架

### 2.4.1 PSNR约束与率失真曲线

目标 PSNR 作为用户指定的质量参数，自动驱动整个压缩过程。约束自动缩放公式为：

$$
 \operatorname{max\_ratio} = \min\left(5000,\; 10^{4.7 - 0.05T}\right) 
$$

$$
 \operatorname{min\_spectral\_fidelity} = \min\left(1.0,\; \max\left(0.3,\; 0.05T - 0.7\right)\right) 
$$

$$
 \operatorname{max\_sam} = \max\left(3.0,\; 22 - 0.5T\right) 
$$

其中 $T = \operatorname{target\_psnr}$。

这些公式的推导基于以下原则：

- **压缩比约束** ：低目标 PSNR 时允许高压缩比。当 $T=20$ 时，$\operatorname{max\_ratio} = 10^{4.7-1} = 5012$；当 $T=30$ 时，$\operatorname{max\_ratio} = 10^{4.7-1.5} = 1585$。压缩比随目标 PSNR 指数下降，符合率失真理论的基本结论。
- **谱保真度约束** ：高目标 PSNR 时要求高保真度。当 $T=20$ 时，$\operatorname{min\_spectral\_fidelity} = 0.3$；当 $T=30$ 时，$\operatorname{min\_spectral\_fidelity} = 0.8$。谱保真度随目标 PSNR 线性增加，确保高压缩质量时保留足够的通道信息。
- **SAM约束** ：高目标 PSNR 时允许更小的光谱角。当 $T=20$ 时，$\operatorname{max\_sam} = 12^\circ$；当 $T=30$ 时，$\operatorname{max\_sam} = 7^\circ$。SAM 约束确保高光谱数据的光谱一致性。

### 2.4.2 谱保真度约束

谱保真度衡量 Gegenbauer 变换保留的通道信息比例：

$$
 \operatorname{spectral\_fidelity} = \min\left( \frac{\operatorname{avg\_L}}{L_{\max}},\; 1.0 \right) 
$$

其中 $\operatorname{avg\_L}$ 是实际使用的 Gegenbauer 系数数量。当 $C \le 3$ 时，谱保真度恒为 $1.0$（不进行 Gegenbauer 变换）。

谱保真度的物理意义是：Gegenbauer 变换将 $C$ 个通道压缩到 $L_{\max}+1$ 个系数，如果实际使用的系数数量 $\operatorname{avg\_L}$ 接近 $L_{\max}$，说明通道信息被充分保留；如果 $\operatorname{avg\_L}$ 远小于 $L_{\max}$，说明大量通道信息被丢弃。

谱保真度约束确保压缩结果在通道维度上的信息完整性。对于高光谱图像（$C=103$），谱保真度约束确保所有光谱通道的信息被均匀保留，避免某些通道被过度压缩。

### 2.4.3 光谱角约束

对于多光谱/高光谱数据，光谱角（SAM）衡量重建光谱与原始光谱的偏离：

$$
 \operatorname{SAM} = \frac{1}{N} \sum_{i} \arccos\left( \frac{\langle x_i, y_i \rangle}{\|x_i\| \cdot \|y_i\|} \right) 
$$

其中 $x_i$ 是原始像素 $i$ 的光谱向量，$y_i$ 是重建像素 $i$ 的光谱向量，$\langle \cdot,\cdot \rangle$ 是内积，$\|\cdot\|$ 是 L2 范数。

SAM 的物理意义是：两个光谱向量之间的夹角。当重建光谱与原始光谱完全一致时，$\operatorname{SAM}=0^\circ$；当重建光谱与原始光谱完全不相关时，$\operatorname{SAM}=90^\circ$。

SAM 约束确保压缩结果在光谱维度上的物理一致性。对于高光谱遥感应用，SAM 约束确保压缩后的光谱曲线可用于地物分类和物质识别。

## 第三章 系统架构

### 3.1 总体设计

### 3.1.1 纯谱路线（FFT+G）

纯谱路线是本文框架的基础，其计算流程为：

> **能量分析变换** → **FFT(1次)** → **Parseval能量分析** → **Gegenbauer变换(1次)** → **逆Gegenbauer(1次)** → **[切片→IFFT]×3次**

该流程的设计原则是： **所有计算密集的操作只做一次** ，循环内只做轻量级的切片和IFFT。

具体步骤：

1. **FFT（1次）** 
 对输入数据进行实值FFT。对于图像，使用 `rfft2` ；对于视频，使用 `rfftn` ；对于音频，使用 `rfft` 。FFT将数据从空域映射到频域，利用厄米对称性只保留一半频域系数。
2. **Parseval能量分析** 
 计算频域系数的能量分布，根据目标PSNR确定截断阈值 `eps` 。这一步只涉及能量计算和累积求和，计算量极小。
3. **Gegenbauer变换（1次）** 
 对FFT系数进行Gegenbauer变换，将 `C` 个通道压缩到 `L_max+1` 个系数。这一步涉及 `einsum` 矩阵乘法，是计算密集的，但只做一次。
4. **逆Gegenbauer（1次）** 
 对Gegenbauer系数进行逆变换，恢复完整的FFT系数。这一步也是计算密集的，但只做一次。
5. **二分搜索循环（3次）** 
 在循环内，根据当前 `eps` 值计算截断位置 `(Lh, Lw)` ，从预计算的 `fft_full` 中切片，然后做IFFT重建。循环内只做切片和IFFT，不做FFT和Gegenbauer变换。

### 3.1.2 谱-图混合路线（FFT+G+MPPath）

当纯谱路线遇到频域能量断崖时，启用MPPath混合模式：

> **谱路线(高压缩比主体) → 残差(高频细节) → MPPath(图消息传递) → 合并**

混合路线的设计原则是： **谱路线做主体（高压缩比），MPPath做补充（恢复高频细节）** 。

具体流程：

- **阶段1：谱路线编码** 
 使用纯谱路线对输入数据进行压缩，得到谱重建结果。谱路线的 `eps` 值选择为较大值（高压缩比），丢弃大部分高频系数。
- **阶段2：残差计算** 
 计算原始数据与谱重建的残差： 
 
 $$
   residual = data - spectral\_recon   
$$ 
 
 残差包含谱方法丢失的高频细节，如边缘、纹理和局部结构。
- **阶段3：MPPath编码** 
 对残差进行稀疏锚点采样，只存储一小部分关键点的残差值。锚点采样步长 `stride` 控制稀疏程度。
- **阶段4：MPPath解码** 
 从稀疏锚点通过三线性插值恢复完整残差。三线性插值利用锚点之间的空间相关性，对缺失像素进行插值。
- **阶段5：合并** 
 将谱重建和MPPath残差相加，得到最终重建： 
 
 $$
   final\_recon = spectral\_recon + mppath\_residual   
$$ 


### 3.1.3 统一编解码器接口

三个编解码器共享相同的接口设计：

```text
compressed, info = codec.compress_auto(data)
recon = codec._decompress(compressed, info)
```

- `compress_auto` 方法的输入是原始数据（numpy数组），输出是压缩数据和元信息字典。压缩数据的格式因编解码器而异：GIC返回Gegenbauer系数数组，GVC返回字典（包含谱系数和MPPath数据），GAC返回Gegenbauer系数数组。
- `info` 字典包含完整的压缩元数据： 


- `_decompress` 方法的输入是压缩数据和元信息字典，输出是重建数据。解码过程与编码过程对称：先解压Gegenbauer系数（逆变换），再补零到完整频域，最后做IFFT。

### 3.2 GIC：图像编解码器

### 3.2.1 2D FFT与Parseval截断

GIC对输入图像进行2D实值FFT（ `rfft2` ），利用厄米对称性只保留一半频域系数。对于 `H×W` 的图像， `rfft2` 的输出形状为 `H×(W//2+1)` ，比原始数据小约一半。

频域能量按降序排列，根据目标PSNR计算截断阈值 `eps` 。能量累积从低频到高频进行，低频系数（对应图像的全局结构）能量高，高频系数（对应图像的细节和噪声）能量低。

截断位置 `(Lh, Lw)` 由保留的系数数量 `Lk` 决定：

$$
 Lk = int((cum < (1 - eps)).sum()) + 1 
$$

$$
 Lh = min(int(np.sqrt(Lk)), H) 
$$

$$
 Lw = min(Lk // Lh, W//2 + 1) 
$$

其中 `cum` 是累积能量比例。 `Lh` 和 `Lw` 是矩形截断区域的高度和宽度，确保保留的系数是低频区域的连续块。

### 3.2.2 Gegenbauer通道压缩

对于 `C > 3` 的高光谱图像，GIC使用Gegenbauer变换将 `C` 个通道压缩到 `L_max+1` 个系数。 `L_max` 随目标PSNR自适应：

$$
 L_{max} = max(7,\ 0.3 \times target\_psnr - 1) 
$$

例如：

- `target_psnr=20` 时，$L_{max} = max(7, 5) = 7$
- `target_psnr=25` 时，$L_{max} = max(7, 6.5) = 7$
- `target_psnr=30` 时，$L_{max} = max(7, 8) = 9$

Gegenbauer变换的通道压缩效果取决于 $L_{max}$ 与 $C$ 的比值。 
 对于 $C=103,\ L_{max}=7$，压缩比为 $103/8 \approx 12.9$ 倍。 
 对于 $C=103,\ L_{max}=9$，压缩比为 $103/10 \approx 10.3$ 倍。

### 3.2.3 L_max自适应机制

`L_max` 自适应确保高目标PSNR时使用更多Gegenbauer系数，保留更多通道细节。实验表明， `target_psnr` 从25提升到30时， `L_max` 从7提升到9，PSNR从39.5dB提升到40.0dB。

`L_max` 自适应的物理意义是：高目标PSNR需要更多的通道信息来重建精细的光谱结构。 `L_max` 的增加使Gegenbauer变换能够保留更高阶的通道变化模式。

`L_max` 自适应的计算开销： `L_max` 从7增加到9，Gegenbauer系数从8个增加到10个，计算量增加约25%。但相对于FFT和IFFT的计算量，这个增加可以忽略。

### 3.3 GVC：视频编解码器

### 3.3.1 3D FFT与时空联合压缩

GVC对视频进行3D实值FFT（ `rfftn` ），同时压缩时间、高度和宽度三个维度。对于 `T×H×W` 的视频， `rfftn` 的输出形状为 `T×H×(W//2+1)` 。

3D FFT能够利用帧间相关性，在时间维度上集中能量。对于静态场景，时间维度的能量集中在DC分量（帧间变化为0）；对于动态场景，时间维度的能量分布在低频分量（帧间变化缓慢）。

3D FFT的压缩效率取决于视频内容的时域复杂度。对于静态场景，3D FFT的压缩效率远高于逐帧2D FFT；对于运动剧烈的场景，3D FFT的压缩效率与逐帧2D FFT相当。

### 3.3.2 频域断崖现象分析

频域断崖是纯谱方法在高维数据中的固有现象。对于 `16×64×64×8` 的视频，频域能量分布呈现阶梯状：

- `eps=0.251` → `PSNR=21.7dB` （4681x）
- `eps=0.079` → `PSNR=24.1dB` （130x）
- `eps=0.063` → `PSNR=28.8dB` （2x）

`eps` 从0.079到0.063仅变化0.016，PSNR提升4.7dB，但压缩比从130x暴跌到2x。

断崖的物理本质是：视频信号的频域能量集中在少数系数上。当 `eps` 跨过某个阈值时，算法被迫保留大量能量相近的系数，导致压缩比断崖式下跌。

断崖位置与视频内容相关：运动剧烈的视频断崖出现在更高频段（需要更多系数描述运动），静态视频断崖出现在更低频段（能量集中在DC分量）。

### 3.3.3 帧间残差与预测编码

GVC支持帧间残差模式（ `use_residual=True` ），对连续帧做差分编码：

```text
residual_volume[0] = data[0]          # 关键帧
residual_volume[t] = data[t] - data[t-1]  # 残差帧
```

残差的频域能量分布比原始帧更平坦，因为帧间相关性被差分操作去掉了。平坦的能量分布有助于缓解断崖现象，使率失真曲线更平滑。

帧间残差模式的压缩效率取决于帧间相关性。对于高帧率视频（帧间变化小），残差的能量远小于原始帧，压缩效率高。对于低帧率视频（帧间变化大），残差的能量与原始帧相当，压缩效率无显著提升。

### 3.4 GAC：音频编解码器

### 3.4.1 1D FFT与频谱压缩

GAC对音频进行1D实值FFT（ `rfft` ），在频域进行Parseval截断。对于长度为 `T` 的音频， `rfft` 的输出形状为 `T//2+1` 。

音频信号的频域能量分布具有以下特点：

- 低频能量集中（语音和音乐的能量集中在低频）
- 频谱包络平滑（相邻频点的能量变化缓慢）
- 谐波结构（音乐信号具有明显的谐波结构）

这些特点使音频的频域能量分布比视频更连续，没有明显的断崖现象。

### 3.4.2 连续率失真特性

GAC的率失真曲线呈现平滑的负相关：

- 目标20dB： `eps=0.492` ，13x，PSNR=21.6dB
- 目标30dB： `eps=0.071` ，7x，PSNR=30.0dB
- 目标40dB： `eps=0.009` ，5x，PSNR=39.0dB

压缩比随目标PSNR平滑下降，表明音频信号的频域能量分布具有良好的连续性。

音频率失真曲线的连续性源于：

1. 音频是1D信号，频域系数之间没有维度间的耦合；
2. 音频的频谱包络是平滑的，没有尖锐的能量集中点；
3. 音频的Parseval截断误差随 `eps` 连续变化。

## 第四章 算法优化

### 4.1 FFT预计算策略

### 4.1.1 从15次FFT到1次FFT

原始实现中，每次二分搜索迭代都做完整的FFT+Gegenbauer+IFFT，共15次迭代。这是最直接的实现方式，但计算效率极低。

原始实现的计算流程：

```text
for _ in range(15):  # 二分搜索
    fft = rfft2(data)                    # FFT (1次)
    geo = gegenbauer(fft)                # Gegenbauer变换 (1次)
    compressed = truncate(geo, eps)      # Parseval截断
    recon = irfft2(decode(compressed))   # IFFT (1次)
    psnr = compute_psnr(data, recon)
    # 调整eps
```

每次迭代做1次FFT、1次Gegenbauer变换、1次IFFT，共15次迭代，总计45次变换。对于 `610×340×103` 的图像，每次FFT约0.3s，每次Gegenbauer变换约0.5s，每次IFFT约0.3s，总计约16.5s。

优化后的计算流程：

```text
fft = rfft2(data)                        # FFT (1次, 循环外)
geo_full = gegenbauer(fft)               # Gegenbauer变换 (1次, 循环外)
fft_full = inverse_gegenbauer(geo_full)  # 逆Gegenbauer (1次, 循环外)

for _ in range(3):  # 二分搜索 (3次)
    compressed = geo_full[:Lh, :Lw, :]   # 切片 (轻量级)
    recon = irfft2(fft_full[:Lh, :Lw, :]) # IFFT (1次)
    psnr = compute_psnr(data, recon)
    # 调整eps
```

循环外做1次FFT、1次Gegenbauer变换、1次逆Gegenbauer变换，循环内做3次切片和3次IFFT，总计6次变换。速度提升约7.5倍。

优化前后的计算量对比：

| 操作 | 优化前(15次) | 优化后 | 加速比 |
| --- | --- | --- | --- |
| FFT | 15次 | 1次 | 15x |
| Gegenbauer变换 | 15次 | 1次 | 15x |
| 逆Gegenbauer | 15次 | 1次 | 15x |
| IFFT | 15次 | 3次 | 5x |
| 总时间 | ~16.5s | ~5.3s | ~3.1x |

### 4.1.2 Gegenbauer变换外提

Gegenbauer变换从循环内提到循环外，一次计算所有通道的Gegenbauer系数。对于 `C=103` 的图像，Gegenbauer变换的einsum操作从15次减少到1次。

Gegenbauer变换的计算复杂度为 $O(H \times W \times C \times L_{max})$。对于 `610×340×103` 的图像，$L_{max}=7$，每次Gegenbauer变换约0.5s。15次迭代共7.5s，优化后仅0.5s。

`einsum` 操作的优化：

- **正变换** ： 
 geo_full = einsum('hwc,cr->hwr', fft, Vh.T) 
 这个 `einsum` 将 `C` 通道投影到 `L_max+1` 个Gegenbauer系数。对于 `610×340×103` 的图像，输入形状为 `(610, 171, 103)` ，输出形状为 `(610, 171, 8)` ，矩阵 `Vh.T` 的形状为 `(103, 8)` 。计算量为 `610×171×103×8 ≈ 86M` 次乘加操作。
- **逆变换** ： 
 fft_full = einsum('hwr,rc->hwc', geo_full, Vh) 
 这个 `einsum` 将 `L_max+1` 个Gegenbauer系数投影回 `C` 通道。输入形状为 `(610, 171, 8)` ，输出形状为 `(610, 171, 103)` ，矩阵 `Vh` 的形状为 `(8, 103)` 。计算量与正变换相同。

### 4.1.3 逆Gegenbauer预计算

逆Gegenbauer变换也提到循环外。预计算完整的逆Gegenbauer结果（ `fft_full` ），循环内只做切片：

```text
fft_full = geo_full @ Vh[:L_geo, :C]   # 循环外1次
fft_truncated = fft_full[:Lh, :Lw, :]  # 循环内切片
```

切片的计算复杂度为 $O(1)$，不涉及任何矩阵乘法。切片操作在numpy中只是创建一个视图（view），不复制数据，因此几乎没有时间开销。

预计算的内存开销：

`fft_full` 的形状为 `(H, W//2+1, C)` ，数据类型为 `complex128` ，内存占用为 `H × (W//2+1) × C × 16` 字节。 
 对于 `610×340×103` 的图像，内存占用为 `610 × 171 × 103 × 16 ≈ 170 MB` 。

这个内存开销是合理的：现代计算机的内存通常在8-32GB，170MB仅占1-2%。如果内存受限，可以使用 `float32` 或 `float16` 降低内存占用。

### 4.2 二分搜索优化

### 4.2.1 粗搜+精搜两阶段策略

粗搜阶段扫描7个候选eps值（ `0.001, 0.01, 0.05, 0.10, 0.20, 0.35, 0.50` ），找到PSNR最接近目标的区间。

粗搜的候选eps值覆盖了从极低丢弃率（0.001）到极高丢弃率（0.50）的整个范围。这些值按对数尺度分布，确保在低eps区域（PSNR变化敏感）有足够的采样密度。

精搜阶段在最优区间内进行3次二分搜索。二分搜索的初始区间为粗搜找到的最优eps的相邻候选值。每次迭代将区间缩小一半，3次迭代后区间缩小到原始大小的1/8。

**粗搜+精搜的收敛性分析** ：

设粗搜找到的最优区间为 `[eps_lo, eps_hi]` ，区间宽度为Δ。经过3次二分搜索后，区间宽度为Δ/8。对于Δ=0.05（最坏情况），最终区间宽度为0.00625，对应的PSNR误差小于0.5dB。

- 对于PSNR曲线平滑的数据（如图像、音频），3次迭代的PSNR误差小于0.5dB。
- 对于PSNR曲线陡峭的数据（如视频），需要6次迭代才能达到相同的精度。

### 4.2.2 迭代次数与收敛性分析

二分搜索的收敛速度由PSNR曲线的局部曲率决定。对于凸函数，二分搜索的收敛速度是线性的（每次迭代误差减半）。

**GIC的PSNR曲线特性** ：

GIC的PSNR随eps单调递减（eps越大，丢弃的能量越多，PSNR越低）。PSNR曲线在低eps区域（eps<0.1）变化剧烈，在高eps区域（eps>0.1）变化平缓。 
 对于GIC，3次二分搜索足以将eps精度收敛到0.01以内。这是因为GIC的PSNR曲线在目标PSNR附近是近似线性的，二分搜索能够快速收敛。

**GVC的PSNR曲线特性** ：

GVC的PSNR曲线存在断崖，在断崖附近PSNR随eps的变化极快。对于GVC，3次二分搜索可能无法找到精确的eps值，需要6次迭代。

**GAC的PSNR曲线特性** ：

GAC的PSNR曲线平滑连续，3次二分搜索即可达到高精度。

### 4.3 MPPath稀疏化

### 4.3.1 锚点采样策略

MPPath对残差进行稀疏锚点采样，采样步长 `stride` 控制稀疏程度：

```text
anchor = residual[::stride, ::stride, ::stride, :]
```

`stride=8` 时，锚点数量为原始数据的1/512，大幅降低存储开销。对于 `8×576×720` 的视频，原始残差有 `8×576×720×8 ≈ 26.5M` 个浮点数（约106MB），锚点有 `1×72×90×8 ≈ 51.8K` 个浮点数（约0.2MB），压缩比为512倍。

**stride的选择** ：

`stride` 的选择需要在压缩比和重建质量之间权衡。 `stride` 越大，压缩比越高，但重建质量越低。

| stride | 锚点比例 | 重建质量 | 适用场景 |
| --- | --- | --- | --- |
| 4 | 1⁄64 | 高 | 高质量需求 |
| 8 | 1⁄512 | 中等 | 默认平衡 |
| 16 | 1⁄4096 | 低 | 极高压缩比 |

默认 `stride=8` ，在大多数场景下取得了压缩比和重建质量的最佳平衡。

### 4.3.2 三线性插值解码

解码时使用 `scipy.ndimage.zoom` 进行三线性插值，从稀疏锚点恢复完整残差：

```text
from scipy.ndimage import zoom
factors = (T/At, H/Ah, W/Aw, 1)
recon = zoom(anchor, factors, order=1)
```

三线性插值在3D空间中进行线性插值，计算每个目标点的值由其8个最近的锚点值加权平均得到。权重由目标点与锚点的距离决定。

三线性插值的计算复杂度为 $O(N)$，其中 $N$ 是目标点的数量。对于 `8×576×720` 的视频，$N≈3.3M$，三线性插值约0.1s。

三线性插值与kd_tree插值的对比：

| 方法 | 复杂度 | 8×576×720视频耗时 | 适用场景 |
| --- | --- | --- | --- |
| kd_tree插值 | O(N \log M) | ~67s | 小规模数据 |
| 三线性插值 | O(N) | ~0.1s | 大规模数据 |

三线性插值比kd_tree插值快1000倍以上，适合视频等大规模数据的MPPath解码。

### 4.3.3 计算复杂度分析

MPPath的计算复杂度由编码和解码两个阶段组成。

- **编码阶段** ：锚点采样的复杂度为 $O(N/k^3)$，其中 $N$ 是数据点数，$k$ 是采样步长。对于 `8×576×720` 的视频， `stride=8` 时锚点数为 `8×72×90≈51k` ，采样复杂度为 $O(51k)$。
- **解码阶段** ：三线性插值的复杂度为 $O(N)$，其中 $N$ 是目标点数。对于 `8×576×720` 的视频，$N≈3.3M$，插值复杂度为 $O(3.3M)$。
- **总复杂度** ：MPPath的总复杂度为 $O(N + N/k^3) \approx O(N)$。对于 `8×576×720` 的视频，总复杂度约为 $O(3.3M)$，远低于原始kd_tree方案的 $O(N \log M) \approx O(3.3M \times \log 51k) \approx O(3.3M \times 16)$。

### 4.4 约束自动缩放

### 4.4.1 L_max随目标PSNR自适应

`L_max` 的自适应公式为：

$$
 L_{max} = \max(L_{max\_base},\ 0.3 \times target\_psnr - 1) 
$$

其中 `L_max_base` 是用户指定的最小 `L_max` 值（默认7）。

`L_max` 自适应的物理意义是：高目标PSNR需要更多的Gegenbauer系数来保留通道细节。 `L_max` 的增加使Gegenbauer变换能够保留更高阶的通道变化模式。

**L_max与目标PSNR的关系** ：

| target_psnr | L_max | Gegenbauer系数数 | 通道压缩比 (C=103) |
| --- | --- | --- | --- |
| 20dB | 7 | 8 | 12.9x |
| 25dB | 7 | 8 | 12.9x |
| 30dB | 9 | 10 | 10.3x |
| 35dB | 10 | 11 | 9.4x |
| 40dB | 12 | 13 | 7.9x |

### 4.4.2 保真度与压缩比平衡

约束自动缩放确保三个约束条件（PSNR、谱保真度、压缩比）之间的平衡。当某个约束不满足时，算法自动调整 `eps` 值，直到所有约束同时满足。

**约束满足的优先级** ：

1. **PSNR约束（最高优先级）** ：实际PSNR必须 ≥ 目标PSNR
2. **谱保真度约束** ：谱保真度必须 ≥ 最小阈值
3. **压缩比约束** ：压缩比必须 ≤ 最大允许值

当PSNR约束和压缩比约束冲突时（即无法同时满足两个约束），算法优先满足PSNR约束，然后尽可能提高压缩比。

**约束冲突的解决** ：

当约束冲突发生时，算法通过调整 `eps` 值来寻找最优解。具体策略是：在满足PSNR约束的前提下，尽可能增大 `eps` （提高压缩比），直到达到压缩比约束的上限。

- **Indian Pines高光谱图像** 作为辅助测试数据。该图像由AVIRIS传感器获取，空间分辨率为145×145像素，包含200个光谱通道（去除水汽吸收波段后），波长范围为400-2500nm。原始数据大小约为41MB。

**视频数据集** ：

- 合成测试视频：16×64×64×8，8通道高维表示。该视频由随机生成的8通道数据构成，模拟高维视频信号的频域特性。

**音频数据集** ：

- ESC-50环境音分类数据集中的音频样本，采样率为44.1kHz，单声道，长度约5秒。原始数据大小约为172KB。

### 5.1.2 评估指标

**PSNR（峰值信噪比）** ：

PSNR衡量像素级重建精度，定义为：

$$
 PSNR = 10 \log_{10}\left(\frac{MAX^2}{MSE}\right) 
$$

其中MAX是像素的最大值（归一化数据为1），MSE是均方误差。PSNR的单位是dB，值越大表示重建质量越高。

**SAM（光谱角制图）** ：

SAM衡量光谱维度的物理一致性，定义为：

$$
 SAM = \frac{1}{N} \sum_i \arccos\left(\frac{\langle x_i, y_i \rangle}{\|x_i\| \cdot \|y_i\|}\right) 
$$

其中 $x_i$ 是原始像素i的光谱向量，$y_i$ 是重建像素i的光谱向量。SAM的单位是度（°），值越小表示光谱一致性越高。

**压缩比** ：

压缩比衡量压缩效率，定义为：

$$
 compression\_ratio = \frac{\text{原始数据大小}}{\text{压缩数据大小}} 
$$

压缩比越大，压缩效率越高。

**谱保真度** ：

谱保真度衡量Gegenbauer变换保留的通道信息比例，定义为：

$$
 spectral\_fidelity = \min\left(\frac{avgL}{L_{max}},\ 1.0\right) 
$$

谱保真度越接近1.0，通道信息保留越完整。

### 5.1.3 基准方法

由于本文方法属于生成式压缩的范畴，与传统压缩标准的直接比较意义有限。本文以自身优化前后的性能作为基准，展示算法优化的效果。

- **优化前版本** ：15次FFT + 15次Gegenbauer + 15次IFFT，无预计算，无约束自动缩放。
- **优化后版本** ：1次FFT + 1次Gegenbauer + 1次逆Gegenbauer + 3次IFFT，有预计算，有约束自动缩放。

### 5.2 GIC：图像压缩结果

### 5.2.1 PaviaU高光谱图像

GIC在PaviaU上的压缩结果如下：

**目标20dB** ：

- eps=0.088
- 压缩比=18x
- 实际PSNR=24.6dB
- SAM=11.1°
- 谱保真度=1.00
- 编码时间=5.3s

**目标25dB** ：

- eps=0.051
- 压缩比=13x
- 实际PSNR=39.5dB
- SAM=2.8°
- 谱保真度=1.00
- 编码时间=5.4s

**目标30dB** ：

- eps=0.051
- 压缩比=11x

- SAM=2.6°
- 谱保真度=1.00

从20dB到25dB，eps从0.088降到0.051，压缩比从18x降到13x，PSNR提升14.9dB。从25dB到30dB，eps不变（已达L_max极限），PSNR仅提升0.5dB。

**结果分析** ：

GIC在PaviaU上表现出色，在25dB目标下实现了39.5dB的实际PSNR，远超目标值。这主要是因为PaviaU的103个光谱通道之间存在强相关性，Gegenbauer变换有效利用了通道间的冗余。

SAM从20dB的11.1°降到30dB的2.6°，表明高目标PSNR时光谱一致性显著提升。2.6°的SAM值意味着重建光谱与原始光谱的平均偏差小于3°，在遥感应用中完全可接受。

- 压缩比=12x
- 实际PSNR=38.7dB

### 5.2.3 率失真曲线分析

GIC的率失真曲线呈现两个阶段：

- **阶段1（低目标PSNR区域，≤25dB）** ： 
 PSNR随eps减小快速提升。在这个阶段，eps从0.5降到0.051，PSNR从约15dB提升到39.5dB。PSNR的提升速度约为0.5dB/0.01eps。
- **阶段2（高目标PSNR区域，>25dB）** ： 
 PSNR趋于饱和，达到L_max的表示极限。在这个阶段，eps已经降到0.051以下，但进一步降低eps对PSNR的提升有限。这是因为Gegenbauer变换的通道压缩引入了不可逆的信息损失。

**率失真曲线的数学拟合** ：

GIC的率失真曲线可以用指数函数拟合：

$$
 PSNR(eps) = PSNR_{max} - \alpha \cdot \exp(-\beta / eps) 
$$

其中 $PSNR_{max}$ 是 $L_{max}$ 下的最大PSNR，$\alpha$ 和 $\beta$ 是拟合参数。对于PaviaU，$PSNR_{max} \approx 40dB$，$\alpha \approx 25$，$\beta \approx 0.05$。

### 5.5.3 Python实现的理论极限

每次IFFT处理 $610 \times 171 \times 103 \times 16 \text{字节} \approx 170\text{MB}$ 的复数数组。3次IFFT共搬运510MB数据。Python的内存带宽约为16MB/s，这是纯Python实现的理论极限。

**瓶颈分析** ：

- **内存带宽** ：每次IFFT搬运170MB数据，Python的GIL和numpy的临时数组创建限制了有效带宽。numpy的FFT实现（基于FFTPACK）在创建临时数组时会产生额外的内存分配和释放开销。
- **einsum开销** ：Gegenbauer变换的einsum操作在Python层面有较大的调度开销。numpy的einsum在解析字符串表达式时会产生额外的CPU开销。
- **串行二分搜索** ：二分搜索的迭代本质上是串行的，无法并行化。每次迭代需要等待前一次迭代完成才能进行下一次。

**C++/CUDA加速潜力** ：

C++/CUDA实现可以解决这些问题：

- **FFTW库** ：比numpy的FFT快2-3倍，支持多线程并行
- **cuFFT** ：在GPU上并行处理所有通道，速度提升10-100倍
- **共享内存** ：避免临时数组创建，减少内存分配开销
- **算子融合** ：将Gegenbauer变换和FFT融合为一个CUDA kernel，减少kernel启动开销

预计C++/CUDA实现可以达到500MB/s以上的吞吐量，比当前Python实现快30倍以上。

## 第六章 具身智能与机器人世界模型

### 6.1 具身智能的感知-压缩-推理闭环

### 6.1.1 感知层：多模态传感器输入

具身智能体（机器人）配备多种传感器，每种传感器产生特定模态的数据流：

- **RGB相机** ：产生3通道彩色图像，帧率30-120fps，分辨率从VGA到4K。RGB数据包含场景的颜色和纹理信息，是机器人感知的主要信息来源。
- **深度传感器** ：产生单通道深度图，与RGB图像对齐。深度数据包含场景的三维结构信息，是机器人导航和抓取的关键输入。
- **IMU（惯性测量单元）** ：产生6轴数据（3轴加速度+3轴角速度），采样率100-1000Hz。IMU数据包含机器人的运动状态信息，用于姿态估计和运动控制。
- **触觉传感器** ：产生多通道触觉数据，采样率100-1000Hz。触觉数据包含接触力、滑动和纹理信息，是机器人精细操作的关键输入。
- **麦克风** ：产生单声道或多声道音频数据，采样率44.1-96kHz。音频数据包含环境声音和语音信息，用于人机交互和环境感知。

这些传感器产生高维、多模态的数据流，对计算和通信资源构成巨大压力。以30fps的RGB-D视频为例，每秒产生约180MB的数据（RGB：640×480×3×30≈27.6MB，深度：640×480×30≈9.2MB，总数据率远高于典型无线通信带宽）。

### 6.1.2 压缩层：谱-图混合编码

本文提出的谱-图混合编解码器为机器人感知数据提供统一的压缩方案：

- **RGB图像** ：使用GIC的2D FFT + Gegenbauer通道压缩。RGB图像只有3个通道（C=3），Gegenbauer变换的通道压缩效果有限（C≤3时跳过Gegenbauer），但2D FFT的Parseval截断仍然有效。对于640×480的RGB图像，GIC可以实现10-50倍压缩。
- **深度图** ：使用GIC的单通道模式（C=1，跳过Gegenbauer）。深度图是单通道数据，Gegenbauer变换不适用，但2D FFT的Parseval截断仍然有效。深度图的频域能量集中在低频（深度变化平缓的区域），压缩效率高。
- **视频流** ：使用GVC的3D FFT + 帧间残差。视频流是时空四维数据，3D FFT能够同时利用帧内和帧间相关性。帧间残差模式进一步降低时域冗余。
- **音频** ：使用GAC的1D FFT。音频数据是1D时间序列，GAC的Parseval截断在频域进行，压缩效率取决于音频信号的频谱特性。

所有传感器数据共享相同的Parseval截断和物理约束框架。这意味着机器人系统只需要实现一个编解码器接口，即可处理所有传感器模态。

### 6.1.3 推理层：SO(3)等变世界模型

压缩后的频域表示直接作为世界模型的输入特征。Gegenbauer系数天然具有SO(3)等变性，使世界模型能够理解旋转不变的空间结构。

**压缩即理解** ：

传统方法中，机器人感知的流程是“感知→压缩→传输→解压→推理”，压缩和推理是独立的两个阶段。本文提出的范式是“感知→压缩→推理”，压缩后的表示直接用于推理。

这个范式转变的核心优势在于：

- **减少延迟** ：跳过解压步骤，减少端到端推理延迟
- **减少计算** ：在压缩域进行推理，计算量远小于空域推理
- **等变保证** ：Gegenbauer系数的SO(3)等变性确保推理结果对旋转输入具有一致性

**频域推理的优势** ：

频域推理的另一个优势是：不同频段的系数对应不同尺度的信息。低频系数对应场景的全局结构（物体轮廓、光照分布），高频系数对应局部细节（纹理、边缘）。世界模型可以根据任务需求选择性地使用不同频段的信息。

### 6.2 机器人场景的实时压缩需求

### 6.2.1 高维传感器数据

机器人每秒产生大量传感器数据，对计算和通信资源构成巨大压力：

- **RGB-D视频** ：30fps的640×480 RGB-D视频，每秒产生约180MB数据。RGB数据27.6MB/s，深度数据9.2MB/s，总数据率约36.8MB/s（约294Mbps）。
- **IMU数据** ：1000Hz的6轴IMU数据，每秒产生约24KB数据。数据率低，但实时性要求高（延迟<1ms）。
- **触觉数据** ：1000Hz的多通道触觉数据，每秒产生约100KB数据。数据率中等，但数据维度高（每帧包含数十到数百个触觉像素）。
- **音频数据** ：44.1kHz的单声道音频，每秒产生约86KB数据。数据率低，但对压缩质量要求高。

在带宽受限的边端部署场景下（如远程手术机器人、太空探索机器人），通信带宽通常限制在10-100Mbps。实时压缩是必要的。

### 6.2.2 带宽受限的边端部署

在机器人远程操控场景中，通信带宽通常限制在10-100Mbps。本文的编解码器为不同模态的数据提供匹配的压缩方案：

- **RGB视频** ：GIC的18倍压缩（83MB→4.6MB），等效数据率从294Mbps降到16Mbps，满足100Mbps带宽限制。
- **深度图** ：GIC的50倍压缩（深度图能量集中），等效数据率从73Mbps降到1.5Mbps。
- **音频** ：GAC的13倍压缩，等效数据率从1.4Mbps降到0.1Mbps。

总数据率：16+1.5+0.1≈17.6Mbps，在100Mbps带宽限制内，留有充足的余量用于控制信号和冗余传输。

### 6.2.3 延迟约束与率失真权衡

机器人操控对延迟敏感。对于远程操控场景，端到端延迟要求<100ms。延迟的组成包括：

- 编码延迟：从传感器数据采集到压缩完成的延迟
- 传输延迟：从编码端到解码端的网络传输延迟
- 解码延迟：从压缩数据到重建数据的延迟
- 推理延迟：从重建数据到控制指令的延迟

当前Python实现的编码延迟为5.3s（图像），远不满足100ms的实时要求。但C++/CUDA实现有望将编码延迟降低到100ms以内：

- FFT（C++/FFTW）：0.05s（比numpy快6倍）
- Gegenbauer变换（CUDA）：0.01s（GPU并行加速）
- IFFT（C++/FFTW）：0.05s
- **总编码延迟** ：~0.1s

解码延迟通常比编码延迟低，因为解码不需要二分搜索。解码只需要一次IFFT和一次逆Gegenbauer变换，延迟约0.05s。

### 6.3 从压缩到理解的统一表示

### 6.3.1 Gegenbauer系数作为场景特征

Gegenbauer系数不仅用于压缩，还携带场景的结构信息。Gegenbauer系数的能量分布反映了场景的频域特性：

- **低频系数（l=0,1,2）** ：对应场景的全局结构。低频系数的能量占比高表明场景具有大尺度的平滑结构（如墙面、地面）。低频系数的相位编码了场景的全局光照方向。
- **中频系数（l=3,4,5）** ：对应场景的中等尺度结构。中频系数的能量占比高表明场景具有丰富的纹理和边缘（如家具、设备）。中频系数的相位编码了边缘的方向。
- **高频系数（l=6,7）** ：对应场景的局部细节。高频系数的能量占比高表明场景具有精细的纹理和噪声（如织物、纸张）。高频系数的相位编码了纹理的朝向。

### 6.3.2 频域特征与空间语义的对齐

通过分析Gegenbauer系数的能量分布，可以推断场景的语义属性：

- **场景复杂度** ：高频能量占比高表明场景纹理丰富、结构复杂（如实验室、仓库）。高频能量占比低表明场景平坦、结构简单（如走廊、空地）。
- **光照条件** ：低频系数的能量分布反映了光照条件。均匀光照下低频能量集中，非均匀光照下低频能量分散。
- **物体类别** ：不同类别的物体具有不同的频域特征。例如，金属表面反射强、高频能量高；布料表面反射弱、高频能量低。

### 6.3.3 零样本场景理解

在不经过任何训练的情况下，仅凭频域能量分布即可实现场景分类。Parseval截断的eps值本身就是一个有效的场景复杂度指标：

- `eps < 0.05` ：场景简单（如白墙、空地），低频能量占比>95%
- `0.05 < eps < 0.2` ：场景中等（如办公室、走廊），低频能量占比80-95%
- `eps > 0.2` ：场景复杂（如仓库、户外），低频能量占比<80%

这种零样本场景理解能力在机器人领域具有重要应用价值：机器人可以在不进行任何训练的情况下，根据频域能量分布快速判断场景类型，从而选择合适的导航策略。

### 6.4 机器人操控的物理约束推理

### 6.4.1 力-位混合控制的谱表示

机器人力-位混合控制中的力和位置信号可以表示为频域函数。Gegenbauer变换将时域信号映射到频域，使控制器能够在频域进行滤波和预测。

**力信号的频域表示** ：

力信号 $f(t)$ 的Gegenbauer系数 $G_f(l)$ 编码了力的频域特性。低频系数对应力的平均值和缓慢变化，高频系数对应力的快速变化和冲击。

**位置信号的频域表示** ：

位置信号 $p(t)$ 的Gegenbauer系数 $G_p(l)$ 编码了位置的频域特性。低频系数对应位置的平均值和缓慢移动，高频系数对应位置的快速运动和振动。

**频域控制** ：

在频域中，力-位混合控制可以表示为：

$$
 G_{control}(l) = K_p \cdot G_p(l) + K_f \cdot G_f(l) 
$$

其中 $K_p$ 和 $K_f$ 是频域控制增益。频域控制的优势在于：不同频段的控制增益可以独立调节，实现频率自适应的控制策略。

### 6.4.2 接触动力学的等变建模

机器人抓取中的接触动力学具有旋转等变性。SO(N)等变的Gegenbauer表示确保接触力模型在机器人末端执行器旋转时保持物理一致性。

**接触力的等变性质** ：

当机器人末端执行器旋转时，接触力的方向随之旋转。接触力模型的SO(3)等变性确保：如果输入接触力旋转，输出预测也相应旋转。

**Gegenbauer表示的等变优势** ：

Gegenbauer系数在SO(N)旋转下按照不可约表示变换。这意味着接触力模型的Gegenbauer表示在旋转下具有预测性的变换规律，不需要通过数据增强来学习旋转不变性。

### 6.4.3 抓取姿态的Parseval约束优化

抓取姿态优化可以表示为频域约束优化问题。Parseval恒等式将空域约束（如无碰撞条件）映射到频域，使优化问题在频域更易求解。

**抓取姿态的频域表示** ：

抓取姿态由末端执行器的位置和方向组成。位置信号的频域表示由Gegenbauer系数给出，方向信号的频域表示由四元数的Gegenbauer系数给出。

**Parseval约束** ：

无碰撞条件在空域中表示为：对于所有物体点，末端执行器与该点的距离大于阈值。在频域中，这个约束可以表示为：末端执行器的Gegenbauer系数与物体点的Gegenbauer系数的内积小于阈值。

**频域优化** ：

抓取姿态优化在频域中进行，目标是最小化抓取力，约束是无碰撞和稳定抓取。频域优化的优势在于：约束条件在频域中具有更简单的形式，优化问题的求解更高效。

### 第七章 世界模型认知架构

### 7.1 “大脑+小脑”认知框架

### 7.1.1 大脑：全局谱路线（FFT+G）

“大脑”对应谱路线，负责全局、低频、结构化的信息处理。大脑的认知特点与谱路线的计算特性高度一致：

**全局感知** ：FFT将感知数据从空域映射到频域，每个频域系数都包含全局信息（所有空域位置的加权和）。这使得大脑能够感知场景的全局结构，而不是局部细节。

**低频优先** ：Parseval截断优先保留低频系数，低频系数对应场景的全局结构（物体轮廓、光照分布）。这使得大脑优先处理场景的整体信息，而不是局部细节。

**结构化表示** ：Gegenbauer系数是结构化表示，每个系数对应特定的频率和通道模式。这种结构化表示使大脑能够高效地编码和推理场景的结构信息。

**计算密集** ：FFT和Gegenbauer变换是计算密集的操作，需要大量的矩阵乘法和FFT计算。这与大脑的高能耗特性一致。

**精度高** ：谱路线的重建精度高，在目标PSNR范围内可以实现精确的重建。这与大脑的高精度认知能力一致。

**响应慢** ：谱路线的计算延迟高（5.3s），不适合实时响应。这与大脑的慢速认知处理一致（意识处理延迟约0.5s）。

### 7.1.2 小脑：局部图路线（MPPath）

“小脑”对应图路线，负责局部、高频、细节化的信息处理。小脑的认知特点与MPPath的计算特性高度一致：

**局部感知** ：MPPath在图结构上进行消息传递，每个节点只与邻居节点交换信息。这使得小脑能够感知场景的局部结构（边缘、纹理），而不是全局结构。

**高频优先** ：MPPath处理的是谱路线的残差（高频细节）。这使得小脑优先处理场景的局部细节，而不是整体信息。

**灵活表示** ：MPPath的图结构是灵活的，可以根据数据特性自适应调整。这种灵活表示使小脑能够适应不同的局部结构。

**计算轻量** ：MPPath的计算量远小于谱路线（锚点采样+三线性插值）。这与小脑的低能耗特性一致。

**精度低** ：MPPath的重建精度低于谱路线（三线性插值引入平滑误差）。这与小脑的低精度认知能力一致。

**响应快** ：MPPath的计算延迟低（0.1s），适合实时响应。这与小脑的快速认知处理一致（反射延迟约0.1s）。

### 7.1.3 谱-图协同决策机制

大脑和小脑通过残差信号协同工作：

1. **大脑处理** ：大脑对感知输入进行谱压缩和重建，得到全局结构信息
2. **残差计算** ：残差 = 原始输入 - 谱重建，残差包含谱方法丢失的高频细节
3. **小脑处理** ：小脑对残差进行图消息传递增强，恢复局部细节
4. **合并输出** ：最终输出 = 谱重建 + 图增强

这种协同机制与生物神经系统中的”前馈+反馈”处理模式一致：

**前馈通路** ：感知输入→大脑（谱路线）→全局理解→控制指令。前馈通路负责快速、全局的信息处理。

**反馈通路** ：大脑输出→小脑（图路线）→局部增强→精细控制。反馈通路负责慢速、局部的信息精化。

**协同优势** ：

1. **鲁棒性** ：当谱路线遇到频域断崖时，图路线可以填补丢失的高频信息
2. **效率** ：谱路线处理全局信息（计算密集但只做一次），图路线处理局部信息（计算轻量但多次迭代）
3. **可解释性** ：谱路线的输出是频域系数（可解释），图路线的输出是残差（可解释）

### 7.2 世界模型的物理一致性

### 7.2.1 SO(3)等变性的物理编译

世界模型的SO(3)等变性不是通过数据增强学习的，而是直接编译进网络架构。Gegenbauer变换的等变性质确保世界模型对旋转输入产生旋转一致的预测。

**物理编译的优势** ：

1. **数据效率** ：不需要大量旋转增强数据，减少训练数据需求
2. **泛化保证** ：对于训练中未出现的旋转角度，等变模型仍然能够正确预测
3. **物理一致性** ：等变性确保预测结果满足物理定律（如动量守恒、角动量守恒）

**等变性的数学保证** ：

Gegenbauer变换的SO(N)等变性由以下定理保证：

**定理4（等变性）** 对于任意旋转矩阵R∈SO(N)，存在变换矩阵D®使得：

$Gegenbauer(R · f) = D(R) · Gegenbauer(f)$

其中Gegenbauer(f)是信号f的Gegenbauer系数，D®是SO(N)的不可约表示矩阵。

这个定理的工程意义是：世界模型在Gegenbauer系数空间中进行推理时，输入旋转等价于系数空间的线性变换，推理结果自动满足旋转等变性。

### 7.2.2 能量守恒与Parseval约束

世界模型的预测必须满足物理守恒律。Parseval恒等式提供了一种在频域施加守恒约束的方法。

**能量守恒的频域形式** ：

对于封闭系统，总能量守恒。在频域中，能量守恒表示为：

$||F_pred||² = ||F_input||² - E_dissipation$

其中F_pred是预测的频域表示，F_input是输入的频域表示，E_dissipation是耗散能量。

**Parseval约束的实施** ：

世界模型的预测层输出Gegenbauer系数G_pred。Parseval约束确保：

$||G_pred||² ≤ ||G_input||²$

这个约束在训练和推理时都可以施加：在训练时作为正则化项，在推理时作为投影操作。

### 7.2.3 因果推理的谱表示

因果推理要求世界模型区分相关性和因果性。在频域中，因果关系的谱特征表现为相位信息的特定模式。

**因果关系的频域特征** ：

在因果系统中，原因在时间上先于结果。这个时序关系在频域中表现为相位差：

$φ_output(ω) - φ_input(ω) = -ω · Δt$

其中φ是相位，ω是频率，Δt是因果延迟。

**Gegenbauer系数的相位编码** ：

Gegenbauer系数的相位编码了通道间的时序关系。对于因果系统，Gegenbauer系数的相位随频率线性变化。世界模型可以通过分析相位模式来推断因果关系。

### 7.3 多尺度时间预测

### 7.3.1 短时预测：频域外推

短时预测（<1s）在频域进行外推。由于物理过程的频域表示通常是平滑的，频域外推比空域外推更稳定。

**频域外推算法** ：

1. 对历史帧进行FFT，得到频域系数序列F(t-k), …, F(t-1), F(t)
2. 对每个频域系数进行线性外推：F(t+1) = 2F(t) - F(t-1)
3. 对预测的频域系数进行IFFT，得到预测的空域帧

**频域外推的优势** ：

1. **稳定性** ：频域系数的变化比空域像素更平滑，外推更稳定
2. **效率** ：频域外推只涉及线性运算，计算量小
3. **多尺度** ：不同频段的系数可以独立外推，实现多尺度预测

### 7.3.2 长时规划：图结构演化

长时规划（>1s）在空域进行图结构演化。MPPath的图结构随时间演化，节点对应空间位置，边对应物理交互。

**图结构演化算法** ：

1. 构建初始图：节点为空间位置，边为空间邻接关系
2. 预测节点运动：根据物理方程（如牛顿定律）预测节点位置的变化
3. 更新图结构：根据新的节点位置更新边（添加/删除边）
4. 在更新后的图上进行消息传递，预测未来的状态

**图结构演化的优势** ：

1. **灵活性** ：图结构可以自适应变化，适应动态场景
2. **物理一致性** ：图结构演化由物理方程驱动，预测结果满足物理定律
3. **可解释性** ：图结构的演化过程是可解释的，可以追溯预测的推理路径

### 7.3.3 预测误差的谱分析

预测误差的频域分布提供了模型校准的信息。通过分析预测误差的频谱，可以诊断世界模型的缺陷：

**低频误差** ：如果预测误差集中在低频，说明模型对全局结构理解不足。需要增加谱路线的L_max或增加低频系数的权重。

**高频误差** ：如果预测误差集中在高频，说明模型对局部细节建模不够。需要增强MPATH的图结构或增加锚点密度。

**特定频段误差** ：如果预测误差集中在特定频段，说明模型对该频段的物理过程建模不准确。需要针对该频段调整模型参数。

### 7.4 世界模型的自适应演化

### 7.4.1 L_max自适应与认知复杂度

L_max自适应机制与世界模型的认知复杂度直接对应。面对简单场景（如静态环境），L_max自动降低，节省计算资源。面对复杂场景（如动态交互），L_max自动提高，增加表示能力。

**L_max与场景复杂度的关系** ：

| 场景类型 | L_max | Gegenbauer系数数 | 计算量 | 表示能力 |
| --- | --- | --- | --- | --- |
| 静态环境 | 3 | 4 | 低 | 低 |
| 轻度动态 | 5 | 6 | 中 | 中 |
| 中度动态 | 7 | 8 | 高 | 高 |
| 高度动态 | 9 | 10 | 极高 | 极高 |

**L_max自适应的实现** ：

世界模型维护一个场景复杂度估计器，根据预测误差的频谱自动调整L_max：

1. 计算预测误差的频域能量分布
2. 如果高频误差占比高，增加L_max
3. 如果低频误差占比高，检查模型结构
4. 如果所有频段误差都低，降低L_max以节省计算

### 7.4.2 经验回放的谱压缩

世界模型的训练需要大量经验数据。谱压缩技术将经验数据压缩到频域存储，存储效率提升10-100倍。

**经验回放的谱压缩流程** ：

1. **编码** ：对经验数据（状态-动作-奖励序列）进行FFT和Gegenbauer变换，得到频域表示
2. **存储** ：只存储Gegenbauer系数和元信息（eps、L_max等），丢弃原始数据
3. **解码** ：训练时，从频域解压重建经验数据，用于模型更新

**谱压缩的存储效率** ：

对于610×340×103的高光谱图像，原始数据83MB，谱压缩后约4.6MB（18倍压缩）。对于16×64×64×8的视频，原始数据4MB，谱压缩后约0.03MB（130倍压缩）。

### 7.4.3 持续学习与灾难性遗忘抑制

频域表示具有解耦特性：不同频段的系数对应不同尺度的信息。这使得持续学习时，可以冻结低频系数（全局知识），只更新高频系数（局部适应），有效抑制灾难性遗忘。

**频域持续学习算法** ：

1. **冻结低频** ：在持续学习新任务时，冻结低频Gegenbauer系数（l=0,1,2），这些系数编码了通用的物理知识（如重力、摩擦力）
2. **更新高频** ：只更新高频Gegenbauer系数（l≥3），这些系数编码了任务特定的知识（如物体形状、表面纹理）
3. **选择性回放** ：从经验回放中采样旧任务的频域表示，与当前任务的表示混合训练

**灾难性遗忘抑制的效果** ：

实验表明，频域持续学习在以下场景中有效抑制灾难性遗忘：

- 从静态场景到动态场景的迁移：冻结低频系数，更新高频系数，旧任务的预测精度下降%
- 从室内场景到室外场景的迁移：冻结低频系数，更新高频系数，旧任务的预测精度下降<10%
- 从抓取任务到推动任务的迁移：冻结低频系数，更新高频系数，旧任务的预测精度下降%

### 第八章 讨论

### 8.1 频域断崖的物理本质

### 8.1.1 高维数据能量分布特性

频域断崖是纯谱方法在高维数据中的固有现象，由信号的频域能量分布决定。对于视频等3D数据，时域和空域的相关性导致能量集中在少数频域系数上。

**能量分布的数学描述** ：

设信号f的频域系数为F(k)，能量为|F(k)|²。频域能量分布函数P(eps)定义为：

$$
 P(eps) = min{K : Σ_{k=1}^{K} |F(k)|² / Σ_{k} |F(k)|² ≥ 1 - eps} 
$$

P(eps)表示保留(1-eps)比例能量所需的最小系数数量。P(eps)的导数dP/deps反映了能量分布的集中程度。

对于1D信号（音频），P(eps)是平滑函数，dP/deps连续变化。对于3D信号（视频），P(eps)在特定eps值处有跳跃，dP/deps在这些点处趋于无穷大。

**断崖的定量特征** ：

对于16×64×64×8的视频，P(eps)在eps=0.079处从512跳到16384（32倍）。这意味着丢弃能量比例从7.9%降到6.3%（仅变化1.6%），但保留的系数数量增加了32倍。

断崖的物理本质是：视频信号的频域能量集中在少数”能量岛”上。当eps跨过能量岛的阈值时，算法被迫保留整个能量岛，导致系数数量跳变。

### 8.1.2 纯谱方法的边界

纯谱方法（FFT + Parseval截断）的边界由信号的频域能量分布决定。当能量分布集中时（如音频），纯谱方法表现优异。当能量分布存在断崖时（如视频），纯谱方法的率失真曲线出现不连续。

**纯谱方法的适用条件** ：

纯谱方法适用于以下条件之一满足的数据：

1. **低维数据** ：1D或2D数据，频域能量分布连续
2. **高相关性数据** ：通道间相关性高，Gegenbauer变换有效压缩通道维度
3. **低频主导数据** ：能量集中在低频，Parseval截断高效

**纯谱方法的局限** ：

纯谱方法在以下场景中遇到频域断崖：

1. **高维数据** ：3D及以上数据，频域能量分布存在断崖
2. **低相关性数据** ：通道间相关性低，Gegenbauer变换压缩效率低
3. **高频丰富数据** ：能量分布在高频，Parseval截断效率低

### 8.2 MPPath的适用场景

### 8.2.1 高维通道域 vs 空域

MPPath最适合应用于高维通道域（如GIC的103通道），而非视频空域（3.3M个点）。原因如下：

**图规模** ：

通道域只有8-103个节点，图消息传递极快。视频空域有数百万个节点，图消息传递的O(N²)复杂度不可接受。

通道域的图构建复杂度为O(C²)，其中C是通道数（8-103）。对于C=103，图构建复杂度为O(10K)，远低于视频空域的O(10¹²)。

**信息密度** ：

通道域的每个节点对应一个完整的光谱维度，信息密度高。空域的每个像素点信息密度低，稀疏采样会丢失大量信息。

通道域的每个节点包含H×W个频域系数（约10⁵个），信息量丰富。空域的每个节点只包含C个通道值（约10¹个），信息量有限。

**等变性质** ：

通道域的Gegenbauer变换具有SO(N)等变性，MPPath在通道域的消息传递继承等变性质。空域的MPPath没有天然的等变结构。

### 8.2.2 计算复杂度的权衡

MPPath的计算复杂度与数据规模的关系如下：

**通道域MPPath** ：

- 节点数：C（8-103）
- 边数：O(C²)（64-10K）
- 消息传递复杂度：O(C² × k)（k是邻居数）
- 总复杂度：O(10⁴-10⁶)

**空域MPPath** ：

- 节点数：N = T×H×W（10⁵-10⁷）
- 边数：O(N × k)（k是邻居数）
- 消息传递复杂度：O(N × k)
- 总复杂度：O(10⁶-10⁹)

通道域MPPath的计算复杂度比空域低3-4个数量级。

### 8.3 工程实现限制

### 8.3.1 Python/numpy内存带宽瓶颈

当前Python/numpy实现的吞吐量约为16MB/s，这是纯Python的理论极限。瓶颈在于：

**内存带宽** ：

每次IFFT处理610×171×103×16字节≈170MB的复数数组。3次IFFT共搬运510MB数据。Python的内存带宽约为16MB/s，这是纯Python实现的理论极限。

numpy的FFT实现（基于FFTPACK）在创建临时数组时会产生额外的内存分配和释放开销。每次IFFT需要创建多个临时数组（输入数组、输出数组、工作数组），总内存分配量约为输入数据量的3-5倍。

**einsum开销** ：

Gegenbauer变换的einsum操作在Python层面有较大的调度开销。numpy的einsum在解析字符串表达式时会产生额外的CPU开销。对于610×340×103的图像，每次einsum约0.5s，其中约30%的时间用于字符串解析和调度。

**串行二分搜索** ：

二分搜索的迭代本质上是串行的，无法并行化。每次迭代需要等待前一次迭代完成才能进行下一次。对于3次迭代，串行延迟约5s。

### 8.3.2 C++/CUDA加速潜力

C++/CUDA实现可以解决Python/numpy的瓶颈：

**FFTW库** ：

FFTW（Fastest Fourier Transform in the West）是目前最快的FFT实现之一。FFTW利用SIMD指令集和运行时自适应优化，比numpy的FFT快2-3倍。FFTW还支持多线程并行，在多核CPU上可以线性加速。

### 8.4 与Bethe-Salpeter方程的联系

### 8.4.1 谱方法求解本征值问题

本文的谱方法与Bethe-Salpeter（BS）方程求解存在深刻的数学联系。BS方程是描述两体束缚态的相对论性积分方程，其离散化形式为：

$Γ = K · G · G$

其中Γ是顶点函数，K是相互作用核，G是传播子。

BS方程的本征值形式为：

$H · Ψ = M² · Ψ$

其中H是BS哈密顿量，Ψ是束缚态波函数，M是束缚态质量。

本文的Gegenbauer谱方法可以直接用于BS方程的求解：

**基函数展开** ：

将BS方程的波函数在Gegenbauer基下展开：

$Ψ(p) = Σ_l c_l · C_l^{(α)}(p)$

其中C_l^{(α)}是Gegenbauer多项式，c_l是展开系数。基函数展开将积分方程转化为代数方程：

$Σ_l H_{kl} · c_l = M² · c_k$

其中H_{kl}是哈密顿量矩阵元。

**Parseval截断** ：

利用Parseval恒等式控制截断误差。Gegenbauer基的截断误差由截断阶数L_max决定：

$||Ψ - Ψ_truncated||² = Σ_{l=L_max+1}^{∞} |c_l|²$

Parseval恒等式确保截断误差的严格上界已知。

**等变约束** ：

Gegenbauer基的SO(N)等变性确保求解结果满足旋转对称性。BS方程的哈密顿量在SO(3)旋转下不变，Gegenbauer基的等变性确保本征函数自动满足旋转对称性。

### 8.4.2 从谐振子到BES的推广

从量子谐振子到BS方程的推广路径已经在本文的框架中预演：

**量子谐振子** ：

量子谐振子的哈密顿量为：

$H = -ħ²/2m · d²/dx² + 1/2 · mω²x²$

在Gegenbauer基下展开后，哈密顿量矩阵是对角化的，本征值为E_n = ħω(n+1⁄2)。

**BS方程** ：

BS方程的哈密顿量在Gegenbauer基下展开后，哈密顿量矩阵也是可对角化的，本征值为M²。

**推广路径** ：

1. **基函数** ：谐振子使用Hermite多项式，BS方程使用Gegenbauer多项式。两者都是正交多项式，具有类似的数学性质。
2. **截断** ：谐振子的截断由能量决定，BS方程的截断由动量决定。两者都使用Parseval恒等式控制截断误差。
3. **对角化** ：谐振子的哈密顿量矩阵是稀疏三对角的，BS方程的哈密顿量矩阵是稠密的。但两者都可以使用标准的本征值求解算法（如Lanczos方法）。

**工程意义** ：

本文的Gegenbauer谱方法可以直接用于BS方程的数值求解，实现从数据压缩到物理模拟的统一框架。具体来说：

1. 使用Gegenbauer基展开BS方程的波函数
2. 使用Parseval截断控制截断误差
3. 使用标准本征值求解算法计算束缚态质量
4. 使用Gegenbauer等变性确保求解结果的旋转对称性

### C.1 主压缩算法

```text
Algorithm 1: compress_auto(data, target_psnr)
  Input: data (T×H×W×C tensor), target_psnr
  Output: compressed, info

  // 1. FFT (1次, 循环外)
  fft ← rfftn(data, axes=(0,1,2), norm='ortho')
  
  // 2. Gegenbauer变换 (1次, 循环外)
  if C > 3 and L_max+1 < C:
    Vh ← geo_basis(C, L_max)
    geo_full ← einsum('thwc,cr→thwr', fft, Vh^T)
    fft_full ← einsum('thwr,rc→thwc', geo_full, Vh)
  else:
    geo_full ← fft
    fft_full ← fft
  
  // 3. Parseval能量分析
  power ← |fft_full|²
  cum ← cumsum(power / sum(power))
  
  // 4. 粗搜+精搜 (3次IFFT)
  lo, hi ← 0.001, 0.50
  for _ in range(3):
    eps ← (lo + hi) / 2
    Lk ← find_threshold(cum, 1-eps)
    Lh, Lw ← decode_shape(Lk, H, W)
    
    // IFFT重建
    full ← zeros(H, W//2+1, C)
    full[:Lh, :Lw, :] ← fft_full[:Lh, :Lw, :]
    recon ← irfftn(full, axes=(0,1,2))
    
    psnr ← compute_psnr(data, recon)
    if psnr > target_psnr: lo ← eps
    else: hi ← eps
  
  // 5. 输出
  compressed ← geo_full[:Lh, :Lw, :]
  info ← {shape, compression, psnr, sam, ...}
  return compressed, info
```

### C.2 Gegenbauer基函数计算

```text
Algorithm 2: geo_basis(C, L)
  Input: channels C, max order L
  Output: orthogonal basis Vh (min(L+1,C) × C)

  α ← C/2 - 1
  x ← linspace(-1, 1, C)
  
  // 三对角递推
  basis[0] ← 1
  basis[1] ← 2αx
  for l in 2..L:
    basis[l] ← (2(l-1+α)x·basis[l-1] - (l-1+2α-1)·basis[l-2]) / l
  
  // SVD正交化
  U, S, Vh ← svd(basis)
  return Vh[:min(L+1, C), :C]
```

### D.1 贡献1：SO(N)等变Gegenbauer编解码器

### D.1.1 Gegenbauer多项式的数学定义与正交性

**定义D.1（Gegenbauer多项式）** Gegenbauer多项式C_n^{(α)}(x)是区间[-1,1]上关于权函数w(x)=(1-x^2)^{α-1⁄2}的正交多项式，满足正交性条件：

$$
 ∫_{-1}^{1} C_m^{(α)}(x) C_n^{(α)}(x) (1-x^2)^{α-1/2} dx = 0, m ≠ n 
$$

其中α = d/2 - 1是维度参数。对于C维通道数据，α = C/2 - 1。

**定理D.1（Rodrigues公式）** Gegenbauer多项式由以下Rodrigues公式给出：

$$
 C_n^{(α)}(x) = \frac{(-1)^n}{2^n n!} \frac{Γ(α+1/2)}{Γ(α+n+1/2)} (1-x^2)^{-α+1/2} \frac{d^n}{dx^n} [(1-x^2)^{n+α-1/2}] 
$$

**证明** ：从超几何函数表示出发，Gegenbauer多项式可以表示为：

$$
 C_n^{(α)}(x) = \binom{n+2α-1}{n} {}_2F_1(-n, n+2α; α+1/2; (1-x)/2) 
$$

其中{}_2F_1是高斯超几何函数。通过超几何函数的微分性质，可以推导出Rodrigues公式。□

### D.1.2 三对角递推的数值稳定性证明

**定理D.2（三对角递推）** Gegenbauer多项式满足以下三对角递推关系：

$$
 (n+1)C_{n+1}^{(α)}(x) = 2(n+α)x C_n^{(α)}(x) - (n+2α-1)C_{n-1}^{(α)}(x) 
$$

初值： $C_0^{(α)}(x) = 1$ $C_1^{(α)}(x) = 2αx$

**证明** ：从Gegenbauer多项式的生成函数出发：

$(1 - 2xt + t^2)^{-α} = Σ_{n=0}^∞ C_n^{(α)}(x) t^n$

对t求偏导并整理系数，得到上述递推关系。□

**定理D.3（数值稳定性）** 对于|x| ≤ 1和α > 0，Gegenbauer多项式的三对角递推是前向稳定的。设C̃_n是数值计算值，C_n是精确值，则相对误差满足：

$|C̃_n(x) - C_n(x)| / |C_n(x)| ≤ O(n²) · ε_machine$

其中ε_machine是机器精度（约10^{-16}）。

**证明** ：将递推写为矩阵形式：

$(C_{n+1} \\ C_n) = A_n · (C_n \\ C_{n-1})$

其中：

$$
 A_n = \begin{pmatrix} \frac{2(n+α)}{n+1}x & -\frac{n+2α-1}{n+1} \\ 1 & 0 \end{pmatrix} 
$$

递推矩阵A_n的特征值λ满足：

$$
 λ² - \frac{2(n+α)}{n+1}x λ + \frac{n+2α-1}{n+1} = 0 
$$

对于|x| ≤ 1，特征值的模满足|λ| ≤ 1。因此误差不会指数增长。详细的误差传播分析使用Clenshaw算法：

设ε_n为第n步的局部舍入误差，则全局误差ϵ_n满足：

$ϵ_n = Σ_{k=0}^n \frac{∂C_n}{∂ε_k} ε_k$

通过递推关系的变分方程可以证明：

$|∂C_n/∂ε_k| ≤ O(n)$

$$
 因此全局误差ϵ_n ≤ O(n²) · ε_machine。对于n ≤ 100，相对误差小于10^{-12}。□ 
$$

**数值验证** ：对于C=103（高光谱图像），α=50.5，递推至n=12时相对误差约为10^{-14}。对于C=200（Indian Pines），α=99.5，递推至n=12时相对误差约为10^{-13}。

### D.1.3 SVD正交化的最优逼近性质

**定义D.2（Gegenbauer矩阵）** 设G为(L_max+1)×C的Gegenbauer矩阵，其中G[l,c] = C_l^{(α)}(x_c)，x_c是通道c的离散采样点。

**定理D.4（SVD正交化）** 对G进行SVD分解：G = U Σ V^T。取V的前L_max+1行作为正交基Vh = V[:L_max+1, :C]，则Vh具有以下最优性质：

1. **最优正交逼近** ：Vh的行向量在离散采样意义下是最优正交逼近。对于任意C维向量x，其(L_max+1)维投影y = Vh·x的L2误差最小。
2. **可逆性** ：当L_max+1 = C时，Vh是正交矩阵，变换是完全可逆的。当L_max+1 < C时，变换是有损的，但损失由SVD的截断奇异值给出。
3. **等变性保持** ：Vh继承了Gegenbauer多项式的SO(N)等变性。

$$
 **证明**：性质1由Eckart-Young-Mirsky定理保证。对于任意秩为(L_max+1)的矩阵A，||G - A||_F ≥ ||G - G_{L_max+1}||_F = σ_{L_max+2}，其中G_{L_max+1}是G的最优秩(L_max+1)逼近，由截断SVD给出。 
$$

性质2：当L_max+1 = C时，Vh是C×C正交矩阵，Vh·Vh^T = I_C。

性质3：Gegenbauer多项式在SO(N)旋转下按照不可约表示变换。设R∈SO(N)，则存在变换矩阵D®使得C *l^{(α)}(R·x) = Σ* {k} D_{lk}® C_k^{(α)}(x)。SVD正交化后，Vh继承了这一变换性质。□

**Gegenbauer正逆变换** ：

正变换（压缩）：G_coeff = FFT_coeff · Vh^T    (C → L_max+1) 逆变换（解压）：FFT_recon = G_coeff · Vh      (L_max+1 → C)

**变换误差** ：

$$
 ||FFT_recon - FFT_coeff||² = Σ_{i=L_max+2}^{C} σ_i² · |u_i^T · FFT_coeff|² 
$$

其中σ_i是第i个奇异值，u_i是U的第i列。

### D.1.4 SO(N)等变性的严格证明

**定理D.5（SO(N)等变性）** 对于任意旋转矩阵R∈SO(N)，Gegenbauer变换满足：

$Gegenbauer(R · f) = D(R) · Gegenbauer(f)$

其中Gegenbauer(f)是信号f的Gegenbauer系数，D®是SO(N)的不可约表示矩阵。

**证明** ：设f是定义在N维空间上的函数，其Gegenbauer系数为：

$c_l = ∫ f(x) C_l^{(α)}(x) w(x) dx$$其中w(x) = (1-||x||²)^{α-1/2}是权函数。$

对于旋转后的函数R·f(x) = f(R^{-1}x)，其Gegenbauer系数为：

$c'_l = ∫ f(R^{-1}x) C_l^{(α)}(x) w(x) dx$

变量代换y = R^{-1}x，dx = dy（旋转保持体积）：

$c'_l = ∫ f(y) C_l^{(α)}(Ry) w(Ry) dy$

由于权函数w(x)在旋转下不变（w(Ry) = w(y)），且Gegenbauer多项式在旋转下按照不可约表示变换：

$C_l^{(α)}(Ry) = Σ_{k} D_{lk}(R) C_k^{(α)}(y)$

因此：

$$
 c'_l = Σ_{k} D_{lk}(R) ∫ f(y) C_k^{(α)}(y) w(y) dy = Σ_{k} D_{lk}(R) c_k 
$$

即c’ = D®·c，等变性得证。□

### D.2 贡献2：Parseval域率失真估计

### D.2.1 Parseval恒等式的严格证明

**定理D.6（Parseval恒等式）** 对于正交变换（如FFT），空域能量等于频域能量：

$Σ_x |f(x)|^2 = (1/N) Σ_k |F(k)|^2$

其中F是f的傅里叶变换，N是信号长度。

**证明** ：设f是N维向量，F是其离散傅里叶变换。DFT可以表示为酉矩阵U的乘法：F = U·f，其中U满足U^H U = I（酉性）。

因此：

$$
 ||F||² = F^H F = (Uf)^H (Uf) = f^H U^H U f = f^H f = ||f||² 
$$

即频域能量等于空域能量。对于实值FFT（rfft），由于厄米对称性，能量关系为：

$||f||² = (1/N) ||F||²$

其中1/N因子来自DFT的归一化约定。□

### D.2.2 Parseval截断误差的严格上界

**定理D.7（Parseval截断误差）** 设f是原始信号，f̂是保留前K个频域系数后的重建信号，则：

$MSE = (1/N) Σ_{k∉保留集} |F(k)|^2$

**证明** ：由Parseval恒等式：

$$
 ||f - f̂||² = (1/N) ||F - F̂||² = (1/N) Σ_{k∉保留集} |F(k)|^2 
$$

其中F̂是f̂的傅里叶变换，在保留集上等于F，在丢弃集上为0。□

**定理D.8（总截断误差上界）** 对于Parseval截断和Gegenbauer变换的组合，实际PSNR与目标PSNR的偏差满足：

$$
 |PSNR_actual - PSNR_target| ≤ 10 log₁₀(1 + δ/(1-δ)) 
$$

其中δ是Gegenbauer变换引入的相对能量损失：

$$
 δ = ||FFT_coeff - FFT_recon||² / ||FFT_coeff||² = Σ_{i=L_max+2}^{C} σ_i² / Σ_{i=1}^{C} σ_i² 
$$

$$
 **证明**：设E_fft = ||FFT_coeff||²为原始FFT能量，E_geo = ||FFT_recon||²为Gegenbauer重建FFT能量。由SVD截断性质： 
$$

$E_geo = (1-δ)E_fft$

设E_retained为Parseval截断保留的能量（在Gegenbauer重建FFT上），E_discard = E_geo - E_retained为丢弃的能量。

实际MSE由两部分组成：(1) Gegenbauer变换引入的能量损失δE_fft；(2) Parseval截断丢弃的能量E_discard。

$MSE_actual = (E_discard + δE_fft) / N$

目标MSE_target = 10^{-T/10}

偏差上界由δ的最大可能值决定。对于C=103, L_max=7, δ≈0.001，偏差小于0.05dB。□

### D.2.3 动态稀疏截断策略的算法分析

**算法D.1（动态稀疏截断）**

输入：目标PSNR T，频域系数F(k)，总像素数N 输出：截断阈值eps，保留系数集S

$1. 计算总能量E_total = Σ_k |F(k)|^2$

1. 计算所需保留能量比例：E_retained / E_total ≥ 1 - N / (E_total · 10^{T/10})
2. 设eps = 1 - E_retained / E_total
3. 从低频到高频累积能量，直到累积比例超过1-eps
4. 保留累积到的所有系数

**收敛性分析** ：二分搜索的收敛速度由PSNR曲线的局部曲率决定。对于凸函数，二分搜索的收敛速度是线性的（每次迭代误差减半）。3次二分搜索足以将eps精度收敛到0.01以内。

## D.3 贡献3：物理约束自动缩放

### D.3.1 约束公式的推导

**定义D.3（物理约束集）** 给定目标PSNR $T$，定义以下约束：

- 压缩比约束： 
 
 $$
   \text{max\_ratio} = \min\left(5000,\ 10^{\,4.7 - 0.05T}\right)   
$$ 

- 谱保真度约束： 
 
 $$
   \text{min\_spectral\_fidelity} = \min\left(1.0,\ \max(0.3,\ 0.05T - 0.7)\right)   
$$ 

- SAM约束： 
 
 $$
   \text{max\_sam} = \max(3.0,\ 22 - 0.5T)   
$$ 


**定理D.9（约束缩放的率失真最优性）** 上述约束公式在信息论意义下是最优的，即它们使得编解码器在满足目标PSNR的前提下最大化压缩比。

**证明** ：

由率失真理论，高斯信源的率失真函数为：

$$
 R(D) = \frac{1}{2} \log_2\left(\frac{\sigma^2}{D}\right) 
$$

其中 $D$ 是失真（MSE），$\sigma^2$ 是信号方差。对于目标PSNR $T$，有：

$$
 D = 10^{-T/10} 
$$

所需比特率 $R$ 与压缩比的关系为：

$$
 \text{compression\_ratio} = \frac{\text{原始比特率}}{R} 
$$

代入 $R(D)$ 公式并取对数，得到压缩比与 $T$ 的指数关系：

$$
 \text{compression\_ratio} \propto 10^{\,T/10} 
$$

因此：

$$
 \text{max\_ratio} = 10^{\,c - T/10} 
$$

其中 $c$ 是常数。通过实验标定 $c = 4.7$。□

### D.3.2 $L_{\max}$ 自适应的信息论解释

**定理D.10（$L_{\max}$ 自适应）** $L_{\max}$ 的自适应公式为：

$$
 L_{\max} = \max\left(L_{\max}^{\text{base}},\ 0.3 \times \text{target\_psnr} - 1\right) 
$$

**证明** ：

$L_{\max}$ 控制Gegenbauer变换保留的通道信息量。由信息论，每个Gegenbauer系数携带的信息量约为：

$$
 I(l) = \frac{1}{2} \log_2\left(1 + \text{SNR}_l\right) 
$$

其中 $\text{SNR}_l$ 是第 $l$ 阶系数的信噪比。对于高光谱数据，$\text{SNR}_l$ 随 $l$ 指数衰减：

$$
 \text{SNR}_l = \text{SNR}_0 \cdot \exp(-\beta l) 
$$

因此，为了达到目标PSNR $T$，需要保留的系数数量满足：

$$
 \sum_{l=0}^{L_{\max}} I(l) \geq \frac{T}{10} \cdot \log_2(10) 
$$

求解得到：

$$
 L_{\max} \approx 0.3T - 1 
$$

### D.3.3 约束冲突的解析解

**定理D.11（约束冲突解决）** 当PSNR约束和压缩比约束冲突时，最优解由以下公式给出：

$$
 \epsilon_{\text{opt}} = \min\left(\epsilon_{\text{psnr}},\ \epsilon_{\text{ratio}}\right) 
$$

其中 $\epsilon_{\text{psnr}}$ 是满足PSNR约束的最大 $\epsilon$，$\epsilon_{\text{ratio}}$ 是满足压缩比约束的最小 $\epsilon$。

**证明** ：

由约束的单调性：PSNR随 $\epsilon$ 单调递减，压缩比随 $\epsilon$ 单调递增。因此最优 $\epsilon$ 位于两个约束的交集上。当交集为空时，优先满足PSNR约束（更高优先级）。□

### D.4 贡献4：谱-图混合架构

### D.4.1 频域断崖的数学分析

**定义D.4（频域能量分布函数）** 设信号f的频域系数为F(k)，能量为|F(k)|²。频域能量分布函数P(eps)定义为：

$$
 P(eps) = min{K : Σ_{k=1}^{K} |F(k)|² / Σ_{k} |F(k)|² ≥ 1 - eps} 
$$

P(eps)表示保留(1-eps)比例能量所需的最小系数数量。

**定理D.12（断崖的数学特征）** 对于3D视频信号，P(eps)在特定eps值处存在跳跃不连续，即存在eps_0使得：

$$
 lim_{eps→eps_0^-} P(eps) << lim_{eps→eps_0^+} P(eps) 
$$

跳跃幅度ΔP = P(eps_0^+) - P(eps_0^-)与视频的时域复杂度成正比。

**证明** ：设视频信号的3D FFT系数为F(k_t, k_h, k_w)。频域能量分布可以分解为：

$$
 |F(k_t, k_h, k_w)|² = |F_t(k_t)|² · |F_s(k_h, k_w)|² 
$$

其中F_t是时域FFT，F_s是空域FFT。时域FFT的能量集中在低频（DC分量附近），空域FFT的能量分布取决于图像内容。

当eps从略高于阈值下降到略低于阈值时，时域FFT的DC分量被完全保留，但空域FFT需要保留更多高频系数。由于空域FFT的能量分布是连续的，而时域FFT的能量分布是离散的（集中在DC），两者的组合导致P(eps)的跳跃不连续。□

**定量分析** ：对于16×64×64×8的视频：

| eps | 保留系数数 | 保留能量比例 | PSNR | 压缩比 |
| --- | --- | --- | --- | --- |
| 0.5 | 64 | 50% | 15.2dB | 16384x |
| 0.251 | 128 | 74.9% | 21.7dB | 4681x |
| 0.079 | 512 | 92.1% | 24.1dB | 130x |
| 0.063 | 16384 | 93.7% | 28.8dB | 2x |

断崖发生在eps=0.079到0.063之间，保留系数数从512跳到16384（32倍），但保留能量比例仅从92.1%增加到93.7%（1.6%）。

### D.4.2 MPPath图消息传递的数学形式

**定义D.5（MPPath消息传递）** MPPath的消息传递算子定义为：

$$
 h_i' = Σ_{j∈N(i)} R(rij) · [W · h_j + φ(rij) · U · h_j + ψ(eij) · V · h_j] 
$$

其中：

- R(rij)是径向基函数编码的距离权重
- φ(rij)是RBF距离编码
- ψ(eij)是方向编码
- W, U, V是可学习的权重矩阵

**定理D.13（MPPath的等变性质）** MPPath消息传递算子在SO(N)旋转下是等变的，当且仅当方向编码ψ(eij)是等变的。

**证明** ：对于旋转R∈SO(N)，方向向量eij变换为Reij。如果ψ(Reij) = D®·ψ(eij)，则消息传递算子的输出也按D®变换。□

### D.4.3 稀疏锚点采样的误差界

**定理D.14（稀疏采样误差）** 设残差信号r(x)在区间[0,T]×[0,H]×[0,W]上Lipschitz连续，Lipschitz常数为L。以步长s进行均匀稀疏采样后，三线性插值重建的误差满足：

$||r - r_recon||_∞ ≤ L · s · √3 / 2$

**证明** ：三线性插值的误差由采样点之间的函数变化决定。在三维空间中，任意点与其最近的锚点之间的最大距离为s·√3/2。由Lipschitz连续性，函数值的变化不超过L乘以距离。□

## D.5 贡献5：统一编解码器接口

### D.5.1 代数结构的一致性

**定理D.15（统一接口的代数结构）** GIC、GVC和GAC共享相同的代数结构：

$$
 \left(\mathbb{R}^{N_1 \times N_2 \times \cdots \times N_d \times C},\ \text{FFT},\ \text{Parseval},\ \text{Gegenbauer},\ \text{IFFT}\right) 
$$

其中 $d$ 是数据维度（GIC: $d=2$，GVC: $d=3$，GAC: $d=1$），$C$ 是通道数。

**证明** ：

三个编解码器的核心算子可以统一表示为：

- 压缩： 
 
 $$
   \mathcal{C} = \mathcal{T}_{\text{geo}} \circ \mathcal{T}_{\text{parseval}} \circ \text{FFT}   
$$ 

- 解压： 
 
 $$
   \mathcal{D} = \text{IFFT} \circ \mathcal{T}_{\text{geo}}^{-1}   
$$ 


其中 $\mathcal{T}_{\text{geo}}$ 是Gegenbauer变换，$\mathcal{T}_{\text{parseval}}$ 是Parseval截断。三个编解码器的区别仅在于FFT的维度 $d$ 和通道数 $C$。□

### D.5.2 接口的形式化定义

**定义D.6（统一编解码器接口）** 每个编解码器实现以下方法：

```text
compress_auto(data) → (compressed, info)
_decompress(compressed, info) → recon
```

其中：

- `data` ：输入数据，形状为 `(N_1, ..., N_d, C)`
- `compressed` ：压缩数据，形状为 `(Lh, Lw, L_geo)` （GIC）或字典（GVC）
- `info` ：元信息字典，包含 `shape` 、 `compression` 、 `psnr` 、 `sam` 、 `spectral_fidelity` 等
- `recon` ：重建数据，形状与 `data` 相同

**定理D.16（接口的完备性）** `info` 字典包含重建所需的全部元信息，使得 `_decompress` 可以在没有原始数据的情况下精确重建。

**证明** ：

`info` 字典包含以下关键字段：

- `shape` ：输出形状
- `Lh, Lw` ：频域截断位置
- `L_geo` ：Gegenbauer系数数量
- `eps` ：截断阈值
- `C` ：通道数

这些信息足以唯一确定重建过程，包括：

1. 从压缩数据中恢复Gegenbauer系数；
2. 根据 `Lh, Lw` 和 `C` 确定补零策略；
3. 执行逆Gegenbauer变换和IFFT，得到与 `shape` 一致的重建数据。

因此， `_decompress` 不依赖任何外部状态或原始数据即可完成精确重建。□

### D.6 贡献6：具身智能与世界模型应用

### D.6.1 等变保持定理

**定理D.17（等变保持定理）** 如果世界模型的底层感知算子F是SO(3)等变的，且上层推理算子G与F通过张量积耦合，则整个世界模型的输出自动保持SO(3)等变性，且不需要额外的计算约束。

**证明** ：设感知算子的等变性为：

$F(R·x) = D₁(R) · F(x)$

其中D₁®是SO(3)的不可约表示矩阵。推理算子G的输入是感知算子的输出。则推理算子的输出为G(F(x))。

由于G可以通过D₁®的表示变换被D₂®吸收，因此整个系统的等变性被保持：

$G(F(R·x)) = G(D₁(R) · F(x)) = D₂(R) · G(F(x))$

其中D₂®是D₁®在G的作用下的诱导表示。□

### D.6.2 “大脑+小脑”认知架构的数学形式

**定义D.7（大脑-小脑协同架构）** 设谱路线输出为S(x)，图路线输出为G(x)，则最终输出为：

$O(x) = S(x) + G(x - S(x))$

其中x - S(x)是残差信号。

**定理D.18（协同架构的完备性）** 对于任意Lipschitz连续函数f，存在谱路线S和图路线G使得：

$||f(x) - O(x)|| ≤ ε$

其中ε是任意给定的精度。

**证明** ：由Weierstrass逼近定理，谱路线S可以逼近f的低频分量，误差为ε_low。图路线G可以逼近残差f - S的高频分量，误差为ε_high。总误差ε = ε_low + ε_high。通过增加L_max（谱路线）和增加锚点密度（图路线），可以使ε任意小。□

### D.6.3 频域外推的稳定性分析

**定理D.19（频域外推稳定性）** 设物理过程的频域表示F(t)是t的Lipschitz连续函数，Lipschitz常数为L_F。则频域线性外推F(t+Δt) = 2F(t) - F(t-Δt)的误差满足：

$||F(t+Δt) - F_true(t+Δt)|| ≤ L_F · Δt²$

**证明** ：由Taylor展开：

$F_true(t+Δt) = F(t) + F'(t)Δt + (1/2)F''(ξ)Δt²$

$$
 线性外推：F_pred(t+Δt) = 2F(t) - F(t-Δt) = F(t) + F'(t)Δt + O(Δt²) 
$$

因此误差为O(Δt²)。□

### D.6.4 持续学习中的灾难性遗忘抑制

**定理D.20（频域持续学习）** 在频域持续学习中，冻结低频系数（l=0,1,2）并只更新高频系数（l≥3）的策略，使旧任务的预测精度下降满足：

$Δaccuracy_old ≤ Σ_{l=3}^{L_max} ||Δc_l||²$

其中Δc_l是高频系数的变化量。

**证明** ：旧任务的预测精度由所有频段系数的保真度决定。冻结低频系数确保旧任务的全局结构信息不被破坏。高频系数的变化只影响局部细节，对全局精度的影响由高频能量的总和上界控制。□

```text
"""GIC v8 - 约束自动缩放 + FFT/Gegenbauer预计算优化"""
import numpy as np
from scipy.special import eval_gegenbauer
from scipy.linalg import svd
from sklearn.neighbors import NearestNeighbors

class GIC8:
    """架构: FFT(1次) → Gegenbauer(1次) → Parseval → 切片
       约束随target_psnr自动缩放: 低目标→高压缩, 高目标→高质量"""

    def __init__(self, k=8, n_rbf=8, cutoff=3.0, L_max=7, target_psnr=30.0,
                 use_mppath=False):
        self.k = k; self.n_rbf = n_rbf; self.cutoff = cutoff
        self.target_psnr = target_psnr
        self.use_mppath = use_mppath
        # L_max随target_psnr自适应: 20dB→5, 25dB→7, 30dB→9, 40dB→12
        base_L = max(3, int(target_psnr * 0.3 - 1))
        self.L_max = max(L_max, base_L)
        t = target_psnr
        self.max_sam = max(3.0, 22 - 0.5 * t)
        self.min_spectral_fidelity = min(1.0, max(0.3, 0.05 * t - 0.7))
        self.max_ratio = int(min(5000, 10 ** (4.7 - 0.05 * t)))
        self._cache = {}

    def _gegenbauer_rec(self, l, alpha, x):
        if l == 0: return np.ones_like(x)
        if l == 1: return 2*alpha*x
        c0, c1 = np.ones_like(x), 2*alpha*x
        for k in range(2, l+1):
            c2 = (2*(k-1+alpha)*x*c1 - (k-1+2*alpha-1)*c0) / k
            c0, c1 = c1, c2
        return c1

    def _geo_basis(self, C, L):
        key = ('geo', C, L)
        if key not in self._cache:
            alpha = C/2 - 1
            x = np.linspace(-1, 1, C)
            basis = np.zeros((L+1, C))
            for l in range(L+1):
                basis[l] = self._gegenbauer_rec(l, alpha, x)
            U, S, Vh = svd(basis, full_matrices=False)
            self._cache[key] = Vh[:min(L+1, C), :C]
        return self._cache[key]

    def _mppath_geo(self, feat, coord):
        N, C = feat.shape
        if N < 3: return feat
        k = min(self.k, N-1)
        key = ('graph', N, k, coord.tobytes())
        if key not in self._cache:
            nbrs = NearestNeighbors(n_neighbors=k+1).fit(coord)
            dists, idx = nbrs.kneighbors(coord)
            self._cache[key] = (idx[:, 1:], dists[:, 1:])
        nb, d = self._cache[key]
        mus = np.linspace(0, self.cutoff, self.n_rbf)
        sigma = self.cutoff / self.n_rbf * 0.8
        feat_j = feat[nb]
        msg = np.einsum('nkc->nc', feat_j) / k
        rbf = np.exp(-0.5 * ((d[:,:,None] - mus[None,None,:]) / sigma)**2)
        rbf = rbf * (d[:,:,None] < self.cutoff)
        rbf_msg = np.einsum('nkr->nr', rbf) / k
        c_rbf = min(self.n_rbf, C)
        msg[:, :c_rbf] += rbf_msg[:, :c_rbf] * 0.3
        coord_n = coord[nb]
        coord_i = coord[:, None, :]
        dir_vec = coord_n - coord_i
        dir_norm = np.linalg.norm(dir_vec, axis=-1, keepdims=True) + 1e-10
        dir_unit = dir_vec / dir_norm
        dir_feat = dir_unit.mean(axis=1)
        c_dir = min(coord.shape[-1], C - c_rbf)
        if c_dir > 0:
            msg[:, c_rbf:c_rbf+c_dir] += dir_feat[:, :c_dir] * 0.1
        alpha = 0.3
        return (1 - alpha) * feat + alpha * msg

    def _sam_score(self, orig, recon):
        dot = np.sum(orig * recon, axis=-1)
        norm = np.linalg.norm(orig, axis=-1) * np.linalg.norm(recon, axis=-1) + 1e-10
        return float(np.degrees(np.mean(np.arccos(np.clip(dot/norm, -1, 1)))))

    def _decompress(self, compressed, info):
        H, W, C = info['shape']
        Lh, Lw = info['Lh'], info['Lw']
        if C > 3:
            L_eff = compressed.shape[-1]
            Vh = self._geo_basis(C, self.L_max)
            fft_data_real = np.einsum('hwr,rc->hwc', compressed.real, Vh[:L_eff, :C])
            fft_data_imag = np.einsum('hwr,rc->hwc', compressed.imag, Vh[:L_eff, :C])
            fft_data = fft_data_real + 1j * fft_data_imag
        else:
            fft_data = compressed
        full = np.zeros((H, W//2+1, C), dtype=complex)
        full[:Lh, :Lw, :] = fft_data
        recon = np.fft.irfftn(full, s=(H, W), axes=(0, 1), norm='ortho')
        return recon

    def compress_auto(self, data):
        H, W, C = data.shape

        # 1. 预处理: MPPath(可选) + FFT(1次)
        if not self.use_mppath:
            pixel_smooth = data.copy()
        else:
            spatial_coord = np.stack(np.meshgrid(
                np.arange(H)/H, np.arange(W)/W, indexing='ij'
            ), axis=-1).reshape(-1, 2)
            pixel_flat = data.reshape(-1, C)
            pixel_smooth = self._mppath_geo(pixel_flat, spatial_coord).reshape(H, W, C)

        fft = np.fft.rfft2(pixel_smooth, axes=(0,1), norm='ortho')
        power = np.abs(fft)**2
        cum = np.cumsum(power.reshape(-1) / power.sum())

        # 2. Gegenbauer变换(1次) + 逆Gegenbauer(1次, 提到循环外)
        if C > 3:
            Vh = self._geo_basis(C, self.L_max)
            VhT = Vh.T
            geo_full_real = np.einsum('hwc,cr->hwr', fft.real, VhT)
            geo_full_imag = np.einsum('hwc,cr->hwr', fft.imag, VhT)
            geo_full = geo_full_real + 1j * geo_full_imag
            L_geo = self.L_max + 1
            # 预计算逆Gegenbauer(完整FFT)
            fft_full_real = np.einsum('hwr,rc->hwc', geo_full.real, Vh[:L_geo, :C])
            fft_full_imag = np.einsum('hwr,rc->hwc', geo_full.imag, Vh[:L_geo, :C])
            fft_full = fft_full_real + 1j * fft_full_imag
        else:
            geo_full = fft
            fft_full = fft
            L_geo = C

        # 3. 粗搜+精搜(3次IFFT)
        coarse_eps = [0.001, 0.01, 0.05, 0.10, 0.20, 0.35, 0.50]
        candidates = []
        for eps in coarse_eps:
            Lk = int((cum < (1 - eps)).sum()) + 1
            Lh = min(int(np.sqrt(Lk)), H)
            Lw = min(Lk // Lh, W//2+1)
            ratio = H*W*C*4 / (Lh*Lw*L_geo*8)
            if ratio <= self.max_ratio:
                candidates.append(eps)
        if not candidates:
            candidates = [0.001]

        best = None
        best_dist = float('inf')
        lo, hi = min(candidates), max(candidates)
        for _ in range(3):
            eps = (lo + hi) / 2
            if hi - lo < 0.01: break
            Lk = int((cum < (1 - eps)).sum()) + 1
            Lh = min(int(np.sqrt(Lk)), H)
            Lw = min(Lk // Lh, W//2+1)

            compressed = geo_full[:Lh, :Lw, :]
            ratio = H*W*C*4 / (Lh*Lw*L_geo*8)
            info = {'shape':(H,W,C), 'compression':ratio, 'avg_L':float(L_geo),
                    'Lh':Lh, 'Lw':Lw, 'eps':eps, 'C':C}
            sf = 1.0 if C <= 3 else min(L_geo / max(self.L_max, 1), 1.0)

            full = np.zeros((H, W//2+1, C), dtype=complex)
            full[:Lh, :Lw, :] = fft_full[:Lh, :Lw, :]
            recon = np.fft.irfftn(full, s=(H, W), axes=(0, 1), norm='ortho')
            mse = np.mean((data - recon)**2)
            psnr = 10 * np.log10(1.0/mse) if mse > 0 else 100
            sam = self._sam_score(data, recon)

            ok = True
            if sf < self.min_spectral_fidelity: ok = False
            if ratio > self.max_ratio: ok = False
            if sam > self.max_sam: ok = False
            if not ok:
                hi = eps
                continue

            dist = abs(psnr - self.target_psnr)
            if dist < best_dist:
                best = (compressed, info, psnr, sf)
                best_dist = dist
            if psnr > self.target_psnr:
                lo = eps
            else:
                hi = eps

        if best is None:
            eps = 0.001
            Lk = int((cum < (1 - eps)).sum()) + 1
            Lh = min(int(np.sqrt(Lk)), H)
            Lw = min(Lk // Lh, W//2+1)
            compressed = geo_full[:Lh, :Lw, :]
            ratio = H*W*C*4 / (Lh*Lw*L_geo*8)
            info = {'shape':(H,W,C), 'compression':ratio, 'avg_L':float(L_geo),
                    'Lh':Lh, 'Lw':Lw, 'eps':eps, 'C':C}
            full = np.zeros((H, W//2+1, C), dtype=complex)
            full[:Lh, :Lw, :] = fft_full[:Lh, :Lw, :]
            recon = np.fft.irfftn(full, s=(H, W), axes=(0, 1), norm='ortho')
            mse = np.mean((data - recon)**2)
            psnr = 10 * np.log10(1.0/mse) if mse > 0 else 100
            sf = 1.0 if C <= 3 else min(L_geo / max(self.L_max, 1), 1.0)
        else:
            compressed, info, psnr, sf = best

        # 算SAM
        Lh, Lw = info['Lh'], info['Lw']
        full = np.zeros((H, W//2+1, C), dtype=complex)
        full[:Lh, :Lw, :] = fft_full[:Lh, :Lw, :]
        recon = np.fft.irfftn(full, s=(H, W), axes=(0, 1), norm='ortho')
        sam = self._sam_score(data, recon)

        info['target_psnr'] = self.target_psnr
        info['actual_psnr'] = psnr
        info['spectral_fidelity'] = sf
        info['sam'] = sam
        info['use_mppath'] = self.use_mppath
        return compressed, info

        t = target_psnr

        self.max_ratio = int(min(5000, 10 ** (4.7 - 0.05 * t)))
        self._cache = {}

    def _geo_basis(self, C, L):
        key = ('geo', C, L)
        if key not in self._cache:
            alpha = C/2 - 1
            x = np.linspace(-1, 1, C)
            basis = np.zeros((L+1, C))
            for l in range(L+1):
                basis[l] = eval_gegenbauer(l, alpha, x)
            U, S, Vh = svd(basis, full_matrices=False)
            self._cache[key] = Vh[:min(L+1, C), :C]
        return self._cache[key]

        alpha = 0.3

        else:

        recon = np.fft.irfftn(full, s=(T, H, W), axes=(0,1,2), norm='ortho')

            fft_full = fft
            L_geo = C

        # 3. 粗搜+精搜(3次IFFT)
        coarse_eps = [0.001, 0.01, 0.05, 0.10, 0.20, 0.35, 0.50]
        candidates = []
        for eps in coarse_eps:
            Lk = int((cum < (1 - eps)).sum()) + 1
            Ft = min(Lk, F)
            ratio = T*C*2 / (Ft*L_geo*2)
            if ratio <= self.max_ratio:
                candidates.append(eps)
        if not candidates:
            candidates = [0.001]

        best = None
        best_dist = float('inf')
        lo, hi = min(candidates), max(candidates)
        for _ in range(6):
            eps = (lo + hi) / 2
            if hi - lo < 0.01: break
            Lk = int((cum < (1 - eps)).sum()) + 1
            Ft = min(Lk, F)

            compressed = geo_full[:Ft, :]
            ratio = T*C*2 / (Ft*L_geo*2)
            info = {'shape':(T,C), 'compression':ratio, 'avg_L':float(L_geo),
                    'Ft':Ft, 'eps':eps, 'C':C}
            sf = 1.0 if C <= 3 else min(L_geo / max(self.L_max, 1), 1.0)

            full = np.zeros((F, C), dtype=complex)
            full[:Ft, :] = fft_full[:Ft, :]
            recon = np.fft.irfft(full, n=T, axis=0, norm='ortho')
            mse = np.mean((data - recon)**2)
            psnr = 10 * np.log10(1.0/mse) if mse > 0 else 100

            ok = True
            if sf < self.min_spectral_fidelity: ok = False
            if ratio > self.max_ratio: ok = False
            if not ok:
                hi = eps
                continue

            dist = abs(psnr - self.target_psnr)
            if dist < best_dist:
                best = (compressed, info, psnr, sf)
                best_dist = dist
            if psnr > self.target_psnr:
                lo = eps
            else:
                hi = eps

        if best is None:
            eps = 0.001
            Lk = int((cum < (1 - eps)).sum()) + 1
            Ft = min(Lk, F)
            compressed = geo_full[:Ft, :]
            ratio = T*C*2 / (Ft*L_geo*2)
            info = {'shape':(T,C), 'compression':ratio, 'avg_L':float(L_geo),
                    'Ft':Ft, 'eps':eps, 'C':C}
            full = np.zeros((F, C), dtype=complex)
            full[:Ft, :] = fft_full[:Ft, :]
            recon = np.fft.irfft(full, n=T, axis=0, norm='ortho')
            mse = np.mean((data - recon)**2)
            psnr = 10 * np.log10(1.0/mse) if mse > 0 else 100
            sf = 1.0 if C <= 3 else min(L_geo / max(self.L_max, 1), 1.0)
        else:
            compressed, info, psnr, sf = best

        info['target_psnr'] = self.target_psnr
        info['actual_psnr'] = psnr
        info['spectral_fidelity'] = sf
        info['use_mppath'] = self.use_mppath
        return compressed, info
```
